# 나의 예약 완료 목록 개발 가이드

본 문서는 고객이 예약을 확정한 후, 실제 출발 전까지의 확정된 여정을 확인하는 '나의 예약 완료 목록' 화면에 대한 개발 내용을 담고 있습니다.

## 1. 목록 조회 조건
- **상태값**: `TB_AUCTION_REQ` 테이블의 `REQ_STAT` 값이 `'CONFIRM'`인 내역.
- **날짜 조건**: 출발 일시(`START_DT`)가 **오늘 날짜보다 이후**인 여정들만 노출합니다.
    - SQL 조건 예시: `REQ_STAT = 'CONFIRM' AND START_DT > CURDATE()`

## 2. 주요 기능 및 UI 구성
- **목록 표시**:
    - 예약 확정된 여정 제목, 구간, 날짜, 차량 정보.
    - 확정된 기사님의 정보(성함, 연락처 등) 연동.
- **주요 액션**:
    - **[티켓 확인]**: 확정된 예약의 상세 영수증 또는 티켓 정보를 모달로 확인합니다.
    - **[취소 요청]**: 부득이한 사정으로 예약을 취소해야 할 경우, 취소 사유를력하고 취소 프로세스를 진행합니다.

## 3. 데이터베이스 연동
- **TB_AUCTION_REQ**: 예약 요청 마스터의 상태(`REQ_STAT`) 및 수정 시간 관리.
- **TB_BUS_RESERVATION**: 확정된 기사의 입찰 정보 및 연락처 조회.

## 4. 디자인 가이드
- 예약 리스트 화면과 통일된 레이아웃을 사용하되, 상태 배지를 '예약 확정'으로 강조하여 사용자가 혼동하지 않도록 합니다.
