# 나의 예약 완료 목록 개발 가이드

성공적으로 확정된 프리미엄 버스 여정을 확인하고 관리하는 화면입니다.

## 1. 개요
- **목적**: 결제 및 확정이 완료된 여정을 조회하고, 배정된 기사님의 연락처 및 티켓 상세 정보를 확인합니다.
- **진입 경로**: 고객 메인 대시보드 > [나의 예약 완료 목록] (또는 확정 내역)

## 2. 주요 기능 및 UI 구성
### A. 예약 완료 목록 (Confirmed List)
- **필수 조회 조건**: `TB_AUCTION_REQ` 테이블에서 `DATA_STAT IN ('BIDDING', 'CONFIRM')` 상태인 여정 조회.
- **데이터 조인 (Core Logic)**: 
    - `TB_AUCTION_REQ` (여정 마스터)
    - `TB_AUCTION_REQ_BUS` (요청 차량 정보)
    - `TB_BUS_RESERVATION` (입찰/확정 정보 - **LEFT JOIN**으로 변경하여 견적 중인 상태도 지원)
    - `TB_USER` (기사님 인적 사항)
    - `TB_BUS_DRIVER_VEHICLE` (확정 차량 상세 정보)
- **데이터 매칭 규칙**: `TB_AUCTION_REQ_BUS.REQ_BUS_SEQ`와 `TB_BUS_RESERVATION.REQ_BUS_ID`를 매칭하여 여정 내 각 차량별 확정 정보를 정확히 매핑.
- **주요 표시 항목**:
    - 여정 제목, 출발/도착지, 일시
    - **확정 기사 정보**: 기사님 성함(복호화 필수), 연락처
    - **확정 차량 정보**: 모델명, 차량 번호
    - **최종 확정 금액**: `TB_BUS_RESERVATION.DRIVER_BIDDING_PRICE` 기반

### B. 티켓 상세 모달 (Ticket Detail Modal)
- 기사님 상세 정보 및 차량 사진 확인.
- 긴급 연락처 및 탑승 안내 정보 제공.

## 3. 기술적 요구사항
- **API**: `GET /api/auction/confirmed/:custId`
- **데이터베이스 연동**:
    - `TB_AUCTION_REQ`: 마스터 상태 관리 (`CONFIRM`, `DONE`)
    - `TB_AUCTION_REQ_BUS`: 요청된 버스 타입 정보 (`REQ_BUS_SEQ` 기반)
    - `TB_BUS_RESERVATION`: 확정된 입찰 정보 및 기사 연결
    - `TB_BUS_DRIVER_VEHICLE`: 확정된 차량 상세 스펙

## 4. 최근 수정 사항 (2026-04-27)
- **서버 에러 수정**: `TB_BUS_RESERVATION` 테이블에 존재하지 않는 `BID_SEQ` 컬럼 참조 제거.
- **스키마 매핑 수정**: `TB_AUCTION_REQ_BUS` 테이블의 `REQ_BUS_ID` 참조를 신규 PK인 `REQ_BUS_SEQ`로 변경하여 500 에러 해결.
- **현지화**: 영문 UI 요소를 한글로 현지화하고 '전체 취소' 버튼을 '여행취소'로 명칭 통일.

---
> [!IMPORTANT]
> **데이터 정합성**: 한 여정에 여러 대의 버스가 있을 경우, 각 버스마다 확정된 기사가 다를 수 있습니다. 반드시 `REQ_ID`와 `REQ_BUS_SEQ`를 조합하여 정확한 매칭을 보장해야 합니다.
