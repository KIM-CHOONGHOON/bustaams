# 이용 내역 확인 화면 개발

## 1. 개요
사용자가 과거에 이용 완료했거나 취소된 여정의 기록을 확인하는 화면입니다. (대시보드 내 모달 팝업 형태)

## 2. 조회 기준 (Data Filtering)
'이용 내역'은 다음의 상태값을 가진 데이터만 선별하여 노출합니다.

- **대상 테이블**: `TB_AUCTION_REQ` (여정 마스터)
- **대상 상태값 (`DATA_STAT`)**:
  - `DONE`: 완료된 건 (화면 표시 명칭)
  - `TRAVELER_CANCEL`: 여행자(고객)에 의한 전체 취소
  - `DRIVER_CANCEL`: 기사에 의한 취소 (여정 중단 등)
  - `BUS_CHANGE`: 버스 변경으로 인한 기존 내역 (이력 관리용)
  - `BUS_CANCEL`: 개별 버스 취소건

## 3. 주요 기능 및 로직
- **API 엔드포인트**: `GET /api/auction/total-history/:custId`
- **UI 정책 (탭 필터링)**:
  - **전체 내역**: `'DONE'`, `'TRAVELER_CANCEL'`, `'DRIVER_CANCEL'`, `'BUS_CHANGE'`, `'BUS_CANCEL'` 전체 노출
  - **완료내역**: 오직 `'DONE'` 상태인 내역만 노출
  - **취소내역**: `'TRAVELER_CANCEL'`, `'DRIVER_CANCEL'`, `'BUS_CHANGE'`, `'BUS_CANCEL'` 상태인 내역만 노출
  - 여정 ID(`#000...`)는 화면에서 숨김 처리하며, 금액 라벨은 "여행 금액"으로 표시합니다.
- **정렬**: 등록일시(`REG_DT`) 기준 최신순으로 정렬하여 최근 기록이 상단에 노출되도록 합니다.
- **상태 배지**: 각 상태값에 맞는 컬러 배지를 적용하여 사용자가 여정의 결과를 한눈에 파악할 수 있게 합니다.

## 4. 데이터 정합성
- 모든 ID 참조는 10자리 숫자형 **`CUST_ID`** 및 **`REQ_ID`** 체계를 따릅니다.
- 기사님 성함 등 개인정보는 복호화 과정을 거쳐 사용자에게 노출합니다.
