# 예약 취소 고객 화면 개발 가이드

나의 예약 완료 목록에서 [예약취소] 버튼 클릭 시 나타나는 모달 화면(`CancelReservationModal.jsx`)의 개발 요구사항입니다.

## 0. 나의 예약 완료 목록 조회 (중요)
- **조회 대상**: `TB_AUCTION_REQ` 테이블의 `REQ_STAT` 값이 `'CONFIRM'`인 모든 예약 내역.
- **조회 원칙**: 개별 차량의 취소(`TRAVELER_CANCEL`) 여부와 상관없이, 상위 예약 마스터의 상태가 `'CONFIRM'`이라면 목록에 반드시 도출되어야 합니다.

## 1. 차량 목록 및 데이터 매핑 기준 (중요)
- **차량 상세 목록**: 예약 원장(`TB_AUCTION_REQ`)에 연결된 확정 내역을 표시합니다.
- **데이터 소스 매핑**:
    - **요청 차량 수**: `TB_AUCTION_REQ_BUS` 테이블의 `REQ_BUS_CNT` 컬럼 정보를 사용합니다.
    - **차종 정보**: `TB_BUS_DRIVER_VEHICLE` 테이블에서 해당 버스의 실제 차종 정보를 불러와 표시합니다.
    - **입찰 금액**: `TB_BUS_RESERVATION` 테이블의 `DRIVER_BIDDING_PRICE` 컬럼 값을 정확히 노출합니다. (0원으로 표시되는 문제 해결 필요)
- **취소 상태 표시**: 개별 차량이 취소된 경우(`TRAVELER_CANCEL`), 목록에서 제외하거나 카드에 '취소됨' 표시를 명확히 합니다.

## 2. 취소 처리 로직 (고객 취소 기준)

### 2.1 개별 차량 취소
- **대상**: 특정 차량 카드 내의 [이 차량만 취소] 버튼 클릭 시.
- **데이터베이스 변경 로직**:
    - **TB_BUS_RESERVATION** 테이블: 해당 버스의 입찰 건 `RES_STAT`를 `'TRAVELER_CANCEL'`로 변경.
    - **중요**: 이 과정에서 상위 마스터 테이블(`TB_AUCTION_REQ`)은 수정하지 않습니다. 마스터 상태는 `'CONFIRM'`을 유지해야 목록에서 사라지지 않습니다.
- **영향 범위**: 해당 개별 차량의 예약만 취소 상태로 변경됩니다.

### 2.2 전체 예약 취소
- **대상**: 하단의 [전체 예약 취소하기] 버튼 클릭 시.
- **데이터베이스 변경 로직**:
    - **TB_AUCTION_REQ** 테이블: 해당 예약 마스터의 `REQ_STAT` 상태값을 `'TRAVELER_CANCEL'`로 변경.
    - **부수 효과**: 마스터 상태가 취소로 변경되므로, 나의 예약 완료 목록 조회 조건(`REQ_STAT = 'CONFIRM'`)에 의해 목록에서 자동으로 제외됩니다.
    - **연관 데이터 처리**: 연결된 모든 입찰 건(`TB_BUS_RESERVATION`)의 상태도 일괄 취소 처리가 필요할 수 있습니다.
- **영향 범위**: 해당 전체 여정 예약이 취소됩니다.

## 3. UI/UX 요구사항
- 취소 전 사용자에게 최종 확인 팝업(`confirm`)을 띄워 실수를 방지합니다.
- 취소 완료 후 성공 메시지를 표시하고, 모달을 닫은 뒤 이전 목록 화면을 새로고침합니다.
- 모든 안내 문구는 한자를 사용하지 않고 명확한 한글로 작성합니다.
