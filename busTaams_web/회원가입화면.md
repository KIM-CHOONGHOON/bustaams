# 회원가입 화면 기술 명세서

본 문서는 busTaams 프로젝트의 회원가입 프로세스 및 사용자 식별 체계에 대한 기술적 사양을 정의합니다.

## 1. 사용자 식별 체계 (ID System)

### 1.1 CUST_ID (사용자 고유 식별자)
- **용도**: 시스템 내부에서 사용자를 식별하는 기본 키 (PK)
- **규격**: `VARCHAR(10)` (10자리 숫자 문자열)
- **특징**: 
    - 1부터 시작하여 1씩 증가하는 순차적인 값입니다.
    - 왼쪽은 10자리가 될 때까지 `0`으로 채워집니다 (Zero-padding).
    - 예시: `0000000001`, `0000000002`

### 1.2 USER_ID (로그인 아이디)
- **용도**: 사용자가 로그인 시 직접 입력하는 식별자 또는 SNS 연동 키값
- **규격**: `VARCHAR(256)`
- **특징**: 
    - 암호화하지 않은 **평문**으로 저장됩니다.
    - **SNS 간편가입(네이버, 카카오 등) 시 제공되는 고유 식별자(최대 256자)를 저장**하기 위해 설계되었습니다.
    - 회원가입 시 중복 체크가 필수입니다.
    - 고유한 값이어야 하며, UNIQUE 제약 조건이 설정되어 있습니다.

### 1.3 EMAIL (이메일 주소)
- **용도**: 알림 수신 및 연락용
- **규격**: `VARCHAR(100)`
- **특징**: `USER_ID`와는 별개의 필드로 관리됩니다.

---

## 2. 이미지 및 서명 관리 (Image & Signature)

### 2.1 전자 서명 (Signature)
- **저장 시점**: 회원가입 시 필수 약관 및 계약서 동의와 함께 전송됩니다.
- **저장 위치**: Google Cloud Storage (GCS)
- **GCS 경로 구조**: `https://storage.googleapis.com/bustaams-secure-data/signatures/{FILE_ID}`
    - 예: `https://storage.googleapis.com/bustaams-secure-data/signatures/00000000000000000016`
- **파일 마스터 관리**:
    - **`TB_FILE_MASTER`**: 생성된 서명 파일의 메타데이터를 저장하고 `FILE_ID`를 발급받습니다.
    - **`TB_USER`**: `SIGNATURE_FILE_ID` 컬럼에 `TB_FILE_MASTER`에서 발급받은 ID를 저장합니다.

### 2.2 약관 동의 이력 (Terms History)
- **`TB_USER_TERMS_HIST`**: 고객이 가입 시 동의한 4개 항목(통합 이용약관, 개인정보 수집 및 이용, 마케팅 수신 동의, 여행자 가입 계약서 등)의 이력을 각각 저장합니다.
    - 각 약관 항목별로 데이터가 생성되며, 동일한 `CUST_ID`와 `SIGN_FILE_ID`를 공유합니다.

### 2.3 기타 이미지 (드라이버 전용)
- 프로필 사진 및 자격증 이미지 등은 회원가입 완료 후 별도의 **기사 등록/프로필 설정 화면**에서 처리합니다.
- `TB_USER.USER_IMAGE` 및 `TB_USER.PROFILE_FILE_ID` 컬럼을 활용합니다.

---

## 3. 화면 구성 및 특이사항 (UI Logic)

### 3.1 회원 구분별 동적 필드
- **버스기사 (DRIVER)** 선택 시:
    - 비밀번호 입력란 아래에 **'추천아이디'** 입력 필드가 추가로 표시되어야 합니다.
- **기타 회원 (TRAVELER, SALESPERSON 등)** 선택 시:
    - '추천아이디' 입력 필드는 표시되지 않아야 합니다.

---

## 4. 데이터베이스 매핑 요약 (TB_USER)

| 컬럼명 | 타입 | 설명 |
| :--- | :--- | :--- |
| `CUST_ID` | `VARCHAR(10)` | 사용자 고유 식별 코드 (PK) |
| `USER_ID` | `VARCHAR(256)` | 로그인 ID 또는 SNS 고유 키값 (UNIQUE) |
| `USER_NM` | `VARCHAR(255)` | 사용자 성명 |
| `PASSWORD` | `VARCHAR(255)` | 해싱된 비밀번호 (bcrypt) |
| `EMAIL` | `VARCHAR(100)` | 이메일 주소 |
| `HP_NO` | `VARCHAR(255)` | 휴대폰 번호 |
| `USER_TYPE` | `ENUM` | 회원 구분 (TRAVELER, DRIVER, PARTNER) |
| `RECOM_CODE` | `VARCHAR(20)` | 추천인 아이디 (기사 가입 시 입력) |
| `SNS_TYPE` | `ENUM` | 가입 채널 (NONE, KAKAO, NAVER) |
| `SIGNATURE_FILE_ID`| `VARCHAR(20)` | 서명 파일 식별자 (`TB_FILE_MASTER` 연동) |
| `PROFILE_FILE_ID` | `VARCHAR(20)` | 프로필 사진 파일 식별자 |
| `USER_IMAGE` | `VARCHAR(255)` | 사용자 이미지 경로/명칭 |
| `SMS_AUTH_YN` | `ENUM` | SMS 인증 여부 (Y/N) |
| `JOIN_DT` | `DATETIME` | 회원가입 일시 |
| `USER_STAT` | `ENUM` | 사용자 상태 (ACTIVE, LEAVE, BANNED, TEMPORARY) |
| `MOD_ID` | `VARCHAR(10)` | 최종 수정자 ID (가입 시에는 본인 CUST_ID) |

---

> [!IMPORTANT]
> - `TB_DRIVER_DOCS` 테이블은 더 이상 사용하지 않으며, 모든 서명 정보는 `TB_FILE_MASTER` 및 `TB_USER_TERMS_HIST`를 통해 관리됩니다.
> - 가입 시 생성되는 모든 로그 및 이력의 `REG_ID`, `MOD_ID`는 생성된 `CUST_ID`를 사용합니다.
