# 회원가입 화면 기술 명세서

본 문서는 busTaams 프로젝트의 회원가입 프로세스 및 사용자 식별 체계에 대한 기술적 사양을 정의합니다.

## 1. 사용자 식별 체계 (ID System)

### 1.1 CUST_ID (사용자 고유 식별자)
- **용도**: 시스템 내부에서 사용자를 식별하는 기본 키 (PK)
- **규격**: `VARCHAR(10)` (10자리 숫자 문자열)
- **특징**: 
    - 1부터 시작하여 1씩 증가하는 순차적인 값입니다.
    - 왼쪽은 10자리가 될 때까지 `0`으로 채워집니다 (Zero-padding).
    - 예시: `0000000001`, `0000000002`

### 1.2 USER_ID (로그인 아이디)
- **용도**: 사용자가 로그인 시 직접 입력하는 식별자 또는 SNS 연동 키값
- **규격**: `VARCHAR(256)`
- **특징**: 
    - 암호화하지 않은 **평문**으로 저장됩니다.
    - **SNS 간편가입(네이버, 카카오 등) 시 제공되는 고유 식별자(최대 256자)를 저장**하기 위해 설계되었습니다.
    - 회원가입 시 중복 체크가 필수입니다.
    - 고유한 값이어야 하지만, 더 이상 기본 키(PK)는 아닙니다.

### 1.3 EMAIL (이메일 주소)
- **용도**: 알림 수신 및 연락용
- **규격**: `VARCHAR(100)`
- **특징**: `USER_ID`와는 별개의 필드로 관리되며, 사용자가 입력한 값이 그대로 저장됩니다.

### 1.4 FILE_ID (파일 식별자)
- **용도**: 서명, 이미지 등 모든 업로드 파일의 고유 ID
- **규격**: **20자리 숫자 문자열** (1부터 시작하여 1씩 증가, 왼쪽은 0으로 패딩)
- **예시**: `00000000000000000001`
- **관리**: `TB_FILE_MASTER` 테이블에서 관리됩니다.

---

## 2. 이미지 및 서명 관리 (Image & Signature)

### 2.1 전자 서명 (Signature)
- **저장 시점**: 회원가입 시 필수로 전송됩니다.
- **저장 위치**: Google Cloud Storage (GCS)
- **GCS 경로 구조**: `signatures/YYYYMM/FILE_ID.png`
    - 예: `signatures/202604/00000000000000000123.png`
- **DB 매핑**:
    - `TB_FILE_MASTER`: 파일의 전체 메타데이터(경로, 확장자, 크기 등)가 저장됩니다. (Category: `SIGNATURE`)
    - `TB_USER`: 서명 확인의 편의성을 위해 가입 시 생성된 서명 정보를 다음 컬럼에 기록합니다.
        - **`PROFILE_FILE_ID`**: 생성된 서명의 20자리 `FILE_ID`
        - **`PROFILE_IMG_PATH`**: 생성된 서명의 GCS 전체 URL 또는 상대 경로

### 2.2 기타 이미지 (드라이버 전용)
- 프로필 사진 및 자격증 이미지 등은 회원가입 완료 후 별도의 **기사 등록/프로필 설정 화면**에서 처리합니다.

---

## 3. 데이터베이스 매핑 요약 (TB_USER)

| 컬럼명 | 타입 | 설명 |
| :--- | :--- | :--- |
| `CUST_ID` | `VARCHAR(10)` | 사용자 고유 식별 코드 (PK) |
| `USER_ID` | `VARCHAR(256)` | 로그인 ID 또는 SNS 고유 키값 |
| `USER_NM` | `VARCHAR(256)` | 사용자 성명 |
| `PASSWORD` | `VARCHAR(256)` | 해싱된 비밀번호 |
| `EMAIL` | `VARCHAR(100)` | 사용자가 입력한 이메일 |
| `HP_NO` | `VARCHAR(20)` | 휴대폰 번호 |
| `USER_TYPE` | `VARCHAR(20)` | 회원 구분 (TRAVELER, DRIVER, PARTNER) |
| `SNS_TYPE` | `VARCHAR(20)` | 가입 채널 (NONE, KAKAO, NAVER) |
| `PROFILE_FILE_ID` | `VARCHAR(20)` | 등록한 서명의 FILE_ID |
| `PROFILE_IMG_PATH` | `VARCHAR(512)` | 서명 이미지 저장 경로 |
| `JOIN_DT` | `DATETIME` | 회원가입 일시 |

---

> [!NOTE]
> 비밀번호는 가입 시 `bcrypt`를 사용하여 해싱된 상태로 `PASSWORD` 컬럼에 저장됩니다.
