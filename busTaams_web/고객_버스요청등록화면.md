# 버스 예약 요청 등록 화면 명세 (고객용)

## 1. 개요
고객이 여정 정보를 입력하고 여러 대의 버스에 대한 견적을 요청하는 화면.

---

## 2. 데이터 생성 및 시퀀스 로직

### A. 식별 체계 및 순번 관리
- **마스터 ID 생성**: `TB_AUCTION_REQ` 테이블에 10자리 0-패딩 숫자 형태의 `REQ_ID`를 생성합니다.
- **고객 매핑**: `TRAVELER_ID` 컬럼에 요청한 사용자의 `CUST_ID` 값을 저장합니다.
- **버스 목록 (`TB_AUCTION_REQ_BUS`)**:
  - 생성된 `REQ_ID`를 참조합니다.
  - 추가된 버스 대수만큼 **`REQ_BUS_SEQ`**를 1부터 순차적으로 증가시켜 저장합니다.
- **경유지 목록 (`TB_AUCTION_REQ_VIA`)**:
  - 단일 `REQ_ID` 내에서 입력된 모든 경로 정보에 대해 **`VIA_SEQ`**를 1부터 순차적으로 증가시켜 저장합니다.
- **금액 및 수수료 계산 (`TB_AUCTION_REQ_BUS`)**:
  - `RES_BUS_AMT`: 사용자가 입력한 해당 버스의 요청 금액.
  - `RES_FEE_TOTAL_AMT`: `RES_BUS_AMT` * 0.066 (6.6% 전체 수수료).
  - `RES_FEE_REFUND_AMT`: `RES_BUS_AMT` * 0.055 (5.5% 환급 가능 수수료).
  - `RES_FEE_ATTRIBUTION_AMT`: `RES_BUS_AMT` * 0.011 (1.1% 귀속 수수료).
  - **상태 관리**: `DATA_STAT` 컬럼에는 초기값으로 **`'AUCTION'`**을 저장합니다.
- **경로 기반 비용 산출 (추후 구현)**:
  - `TOLLS_AMT`: 지도 API를 통해 산출된 고속도로 통행료 정보를 저장합니다.
  - `FUEL_COST`: 이동 거리와 버스 연비를 계산하여 유류비를 산출 후 저장합니다.

### B. 디버깅 및 추적성 확보
- **로그 시스템 도입**: 
  - **Front-end**: 브라우저 콘솔에 서버로 전송되는 최종 `Payload`를 출력합니다 (`[DEBUG] Reservation Payload`).
  - **Back-end**: 서버 콘솔에 수신된 `REQ_BODY`와 실제 인서트되는 `VIA`, `BUS` 정보를 단계별로 출력합니다.

## 3. 기술 사양

### 데이터베이스 테이블 및 컬럼
| 테이블명 | 컬럼명 | 설명 |
|---------|-------|------|
| `TB_AUCTION_REQ` | `REQ_ID` | 예약 요청 ID (PK, 10자리 0-패딩 숫자) |
| `TB_AUCTION_REQ` | `TRAVELER_ID` | 요청 고객 식별 코드 (CUST_ID 값) |
| `TB_AUCTION_REQ_BUS` | `REQ_BUS_SEQ` | 버스 순번 (1부터 시작하는 일련번호) |
| `TB_AUCTION_REQ_BUS` | `RES_BUS_AMT` | 해당 버스 요청 금액 (DECIMAL(13,0)) |
| `TB_AUCTION_REQ_BUS` | `RES_FEE_TOTAL_AMT` | 전체 예약 수수료 (금액의 6.6%) |
| `TB_AUCTION_REQ_BUS` | `RES_FEE_REFUND_AMT` | 예약 환급 금액 (금액의 5.5%) |
| `TB_AUCTION_REQ_BUS` | `RES_FEE_ATTRIBUTION_AMT` | 예약 귀속 금액 (금액의 1.1%) |
| `TB_AUCTION_REQ_BUS` | `DATA_STAT` | 데이터 상태 (초기값: 'AUCTION') |
| `TB_AUCTION_REQ_BUS` | `TOLLS_AMT` | 고속도로 통행료 (경로 계산 후 저장) |
| `TB_AUCTION_REQ_BUS` | `FUEL_COST` | 예상 유류비 (거리/연비 기반 산출) |
| `TB_AUCTION_REQ_VIA` | `VIA_SEQ` | 경유지 순번 (1부터 시작하는 일련번호) |
| `TB_AUCTION_REQ_VIA` | `VIA_TYPE` | `START_NODE`, `START_WAY`, `ROUND_TRIP`, `END_WAY`, `END_NODE` |
