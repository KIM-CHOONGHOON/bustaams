## 1. 목록 표시 조건 (REQ_STAT 필터)

### 변경 내용
- **이전**: `TB_AUCTION_REQ.REQ_STAT = 'BIDDING'` 인 경우만 목록에 표시
- **변경 후**: `TB_AUCTION_REQ.REQ_STAT IN ('BIDDING', 'TRAVELER_CANCEL')` 인 경우 목록에 표시

### 적용 위치
- **백엔드**: `GET /api/auction/user/:userUuid` (server.js)
- **쿼리 조건**: `AND r.REQ_STAT IN ('BIDDING', 'TRAVELER_CANCEL')`

## 2. 출발지 / 도착지 표시

### 데이터 출처
출발지와 도착지는 `TB_AUCTION_REQ_VIA` 테이블에서 조회한다.

| 구분 | VIA_TYPE 값 | 설명 |
|------|------------|------|
| 출발지 | `START_WAY` | 여정 출발 지점 |
| 도착지 | `ROUND_TRIP` | 여정 도착 지점 (왕복 기준) |

### 표시 형식
- 전체 주소 중 **"시/도" + "시군구"** (앞 두 단어)만 표시
- 예) `서울특별시 강남구 봉은사로 지하 201` → **`서울특별시 강남구`**
- 예) `경기도 고양시 덕양구 토당로 35` → **`경기도 고양시`**

### 구현 방식

#### 백엔드 (server.js) — 서브쿼리로 VIA 주소 조회
```sql
(SELECT VIA_ADDR FROM TB_AUCTION_REQ_VIA
 WHERE REQ_UUID = r.REQ_UUID AND VIA_TYPE = 'START_WAY'
 ORDER BY VIA_ORD ASC LIMIT 1) AS VIA_START_ADDR,

(SELECT VIA_ADDR FROM TB_AUCTION_REQ_VIA
 WHERE REQ_UUID = r.REQ_UUID AND VIA_TYPE = 'ROUND_TRIP'
 ORDER BY VIA_ORD ASC LIMIT 1) AS VIA_END_ADDR
```

#### 프론트엔드 (CustomerDashboard.jsx) — 주소 파싱 함수
```js
// 앞 두 단어만 추출 (시/도 + 시군구)
const trimAddress = (addr) => {
  if (!addr || typeof addr !== 'string') return '';
  const parts = addr.trim().split(/\s+/);
  return parts.slice(0, 2).join(' ');
};

// VIA 주소 우선, 없으면 START_ADDR / END_ADDR 폴백
const startAddrDisplay = trimAddress(req.VIA_START_ADDR || req.START_ADDR) || '출발지 미정';
const endAddrDisplay   = trimAddress(req.VIA_END_ADDR   || req.END_ADDR)   || '도착지 미정';
```
