## 1. 목록 표시 조건 (DATA_STAT 필터)

### 변경 내용
- **이전**: `TB_AUCTION_REQ.DATA_STAT IN ('AUCTION', 'BIDDING')` 인 경우 목록에 표시
- **변경 후**: `TB_AUCTION_REQ.DATA_STAT NOT IN ('DONE', 'TRAVELER_CANCEL')` 인 경우 목록에 표시 (**'CONFIRM'** 등 진행 중인 모든 상태 포함)

### 적용 위치
- **백엔드**: `GET /api/auction/user/:custId` (server.js)
- **조회 조건**: `TB_AUCTION_REQ.TRAVELER_ID = :custId` (TRAVELER_ID는 사용자의 CUST_ID 값)
- **시간 조건**: `AND r.START_DT > NOW()` (현재 시점 이후의 여정만 노출, 오늘 날짜 제외)
- **쿼리 조건**: `AND r.DATA_STAT NOT IN ('DONE', 'TRAVELER_CANCEL')`
- **카드 구성**: 하나의 `REQ_ID` 당 **1개의 카드**만 노출 (차량이 여러 대인 경우 서브쿼리로 요약 정보를 합쳐서 제공)

## 2. 출발지 / 도착지 표시

### 데이터 출처
출발지와 도착지는 `TB_AUCTION_REQ_VIA` 테이블에서 조회한다. (데이터가 없을 경우 `TB_AUCTION_REQ` 마스터의 `START_ADDR`, `END_ADDR` 사용)

| 구분 | VIA_TYPE 값 | 설명 |
|------|------------|------|
| 출발지 | `START_WAY` | 여정 출발 지점 |
| 도착지 | `ROUND_TRIP` | 여정 도착 지점 (왕복 기준) |

### 표시 형식
- 전체 주소 중 **"시/도" + "시군구"** (앞 두 단어)만 표시
- 예) `서울특별시 강남구 봉은사로 지하 201` → **`서울특별시 강남구`**

### 구현 방식

#### 백엔드 (server.js) — 서브쿼리로 VIA 주소 조회
```sql
(SELECT VIA_ADDR FROM TB_AUCTION_REQ_VIA
 WHERE REQ_ID = r.REQ_ID AND VIA_TYPE = 'START_WAY'
 ORDER BY VIA_ORD ASC LIMIT 1) AS VIA_START_ADDR,

(SELECT VIA_ADDR FROM TB_AUCTION_REQ_VIA
 WHERE REQ_ID = r.REQ_ID AND VIA_TYPE = 'ROUND_TRIP'
 ORDER BY VIA_ORD ASC LIMIT 1) AS VIA_END_ADDR
```

#### 프론트엔드 (CustomerDashboard.jsx) — 주소 파싱 함수 및 ID 매핑
- **API 호출**: `api/auction/user/${user.custId}` (내부 고유 식별자 사용)
- **식별자 매핑**: `req.REQ_ID`를 고유 키 및 선택 로직의 식별자로 사용
