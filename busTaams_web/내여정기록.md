## 내 여정 기록 개발 내역 및 분석 리포트

### 1. 레거시 분석 및 설계 방향
- **참고 파일**: `TripHistory.html` (프리미엄 디자인 기반)
- **분석 결과**: 
    - 여정의 날짜, 제목, 이용 금액을 요약하여 리스트로 보여주는 구조가 핵심.
    - 기존 통계 항목(탄소 절약량 등)은 데이터 불확실성으로 인해 초기 버전에서 제외하기로 결정.
    - 대시보드에서 팝업되는 고해상도 모달 형태로 구현 방침 수립.

### 2. 구현 결과 상세

#### [서버] 신규 API 구축
- **엔드포인트**: `GET /api/auction/history/:userUuid`
- **로직**:
    - `TB_AUCTION_REQ`와 `TB_BUS_RESERVATION` 테이블을 조인.
    - 과거의 모든 예약 신청 건(완료 및 취소 포함)을 신청일 최신순으로 조회.

#### [프론트엔드] React 컴포넌트 개발
- **컴포넌트**: `TripHistoryModal.jsx` (새로 생성)
- **주요 기능**:
    - 여정 상태별(전체, 완료/확정, 취소) 실시간 필터링 탭 적용.
    - 여정명, 출발지->도착지 경로, 최종 이용 금액 표시.
    - 예약 상태에 따른 컬러 뱃지(확정, 이용완료, 취소됨 등) 적용.
- **연동**: `CustomerDashboard.jsx` 내 '이용 내역' 버튼 클릭 시 모달이 나타나도록 버튼 이벤트와 상태 관리 연결 완료.
