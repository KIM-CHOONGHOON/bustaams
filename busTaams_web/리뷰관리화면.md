# 리뷰 관리 (기사 정보 기반 단일 리스트) 화면 개발

## 1. 개요
사용자가 이용 완료된 여정에 대해 기사님 정보를 확인하며 평가를 남기고, 작성된 평가의 상세 이력을 하나의 통합 목록에서 관리하는 화면입니다.

## 2. 주요 구성 요소 (UI/UX)
- **통합 단일 리스트 (Single Stream List)**: 
  - 탭 구분 없이 모든 이용 완료 여정을 하나의 목록으로 노출
  - 각 카드는 기사 정보와 여정 정보를 동시에 포함
- **기사 프로필 노출**:
  - 여정 정보 옆에 해당 여정을 수행한 기사님의 프로필 사진과 성명 배치
  - 평가 대상(인적 요소)을 명확히 인지할 수 있는 UI
- **버튼 시스템**:
  - **평가하기**: 신규 리뷰 작성을 위한 버튼 (기존 '리뷰 작성하기'에서 명칭 변경)
  - **평가상세**: 이미 작성된 평가 내용 및 기사님의 답글을 확인하기 위한 버튼
- **인라인 상세 영역 (Expandable Detail)**:
  - '평가상세' 클릭 시 해당 카드 하단에 작성된 내용과 기사 답글이 펼쳐지는 구조

## 3. 데이터 연동 (DB 및 API 설계)

### 3.1 데이터베이스 매핑 (`TB_TRIP_REVIEW`)
- **테이블명**: `TB_TRIP_REVIEW` (여정 평가 및 답글 저장)
- **주요 컬럼 및 저장 규칙**:
  - `REQ_ID`: 해당 여행(경매 요청) 아이디
  - `RES_ID`: `TB_BUS_RESERVATION` 테이블의 키 (특정 기사/차량 식별용)
  - **작성자 구분 로직**:
    - **여행자 평가 시**: `WRITER_ID` 컬럼에 여행자의 `CUST_ID` 저장
    - **운전기사 답글 시**: `DRIVER_ID` 컬럼에 운전기사의 `CUST_ID` 저장
  - `STAR_RATING`: 여행자가 부여한 별점 (1~5)
  - `REVIEW_CONTENT`: 평가 및 답글 본문
  - `REG_DT`: 작성 일시

### 3.2 주요 API 엔드포인트
- **목록 조회**: `GET /api/review/total-list/:custId`
  - `TB_AUCTION_REQ`, `TB_BUS_RESERVATION`, `TB_TRIP_REVIEW` 조인하여 데이터 구성
- **평가 저장**: `POST /api/review/submit`
  - 여행자의 신규 평가 데이터를 `TB_TRIP_REVIEW`에 `INSERT`
- **답글 저장**: `POST /api/review/reply`
  - 기사의 답글 데이터를 해당 여정의 리뷰 레코드에 업데이트 또는 신규 저장
- **휴먼 중심 UI**: 기사님의 얼굴과 이름을 강조하여 서비스의 신뢰도와 인적 소통 강조
- **직관적 버튼**: '평가하기'와 '평가상세'를 나란히 배치하여 상태와 상관없이 접근성 유지
- **에디토리얼 감성**: 깨끗한 화이트 배경과 Teal 포인트 컬러의 세련된 조화
