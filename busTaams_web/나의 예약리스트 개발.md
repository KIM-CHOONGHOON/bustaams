# 나의 예약리스트 개발 가이드 (최신 요건 통합본)

본 문서는 고객 대시보드의 '나의 예약목록' 화면에서 수행되는 기사 변경, 버스 취소 및 견적 확정 로직에 대한 최종 개발 명세입니다.

## 1. 개요
- **목적**: 사용자가 등록한 예약 요청을 관리하고, 기사들의 견적을 비교하여 확정하거나 변경/취소합니다.
- **주요 변경점**: UUID 체계를 완전히 제거하고 10자리 숫자 ID(`REQ_ID`, `RES_ID`) 기반의 시스템으로 전환되었으며, 기사 변경과 버스 취소의 로직이 명확히 분리되었습니다.

## 2. 주요 기능 및 UI 구성

### A. 예약 내역 목록 화면 (Main Screen)
- **레이아웃**: 오버레이 모달(Modal) 방식, 여정(`REQ_ID`)별 차량 그룹화 표시.
- **주요 액션 버튼**:
    - **[기사 변경]**: (구 버스 취소) 차량 요청은 유지하되, 현재 확정된 기사를 취소하고 다시 경매(`AUCTION`)를 진행.
        - **노출 조건**: 해당 차량에 응찰/확정된 기사가 있는 경우 표시.
        - **제한**: 여정당 **최대 1회**만 가능 (`BUS_CHANG_CNT` 관리).
        - **알림**: 취소된 기사님께 "고객 요청으로 취소됨" 알림톡 발송.
    - **[버스 취소]**: (구 버스 변경) 여정 내 특정 차량 요청 자체를 영구적으로 삭제.
        - **노출 조건**: 항상 표시.
        - **특이사항**: 마지막 남은 1대를 취소할 경우, 마스터 여정 전체가 **`'TRAVELER_CANCEL'`** 처리됨.
    - **[여행 취소]**: 여정 전체를 취소. (상태값이 **`'AUCTION'`**, **`'BIDDING'`**인 경우만 가능)

### B. 견적 현황 리스트 및 기사 상세 (Quotation & Detail)
- **견적 리스트**: 최신 등록순으로 기사님들의 제안 금액 표시.
- **기사 상세 정보**: **`QuotationDetail`** 컴포넌트를 활용한 모달 레이어링.
    - 기사 프로필(성함, 사진, 경력, 소개글), 차량 정보(모델, 연식, 옵션), 별점 정보 노출.
    - 숫자 ID(`selectedBidId`) 기반으로 데이터를 조회하며, `QuotationDetail.jsx` 원본은 보존함.

## 3. 기술적 로직 및 API 명세

### A. 백엔드 API
- `POST /api/auction/bus-cancel`: **기사 변경 로직**
    - `BUS_CHANG_CNT` 1 증가, 마스터 및 버스 상태를 `'AUCTION'`으로 변경.
    - 기존 예약 데이터를 `'BUS_CHANGE'`로 무효화. (필드명: `reqId`, `reqBusSeq` 사용)
    - 기존 기사님께 알림톡 자동 발송.
- `POST /api/auction/bus-change`: **버스 취소(삭제) 로직**
    - 선택된 버스 상태를 `'TRAVELER_CANCEL'`로 변경.
    - 잔여 버스가 1대뿐이었다면 마스터 상태도 `'TRAVELER_CANCEL'`로 변경.
    - 전체 금액(`REQ_AMT`)에서 해당 차량가 차감.

### B. 데이터베이스 상태값 (Status Codes)
- **`'AUCTION'`**: 경매 진행 중 (기사 변경 후 재경매 포함).
- **`'BUS_CHANGE'`**: 기사 변경으로 인해 무효화된 기존 예약 건.
- **`'TRAVELER_CANCEL'`**: 사용자의 요청으로 취소/삭제된 여정 또는 버스.
- **`'CONFIRM'`**: 최종 예약 확정 상태.

## 4. 데이터베이스 연동 테이블
- `TB_AUCTION_REQ`: 경매 마스터 (`BUS_CHANG_CNT`, `DATA_STAT` 관리)
- `TB_AUCTION_REQ_BUS`: 요청 차량 상세 (`DATA_STAT`, `REQ_BUS_SEQ` 관리)
- `TB_BUS_RESERVATION`: 기사 응찰 및 확정 상태 (매칭 필드: `REQ_ID`, `REQ_BUS_SEQ`)
- `TB_DRIVER_INFO`: 기사 기본 정보 및 알림 발송용 연락처 소스

---
> [!IMPORTANT]
> 모든 ID 변수명에서 **`Uuid`**를 제거하고 **`Id`** (예: `selectedBidId`, `reqId`)를 사용하며, 프론트엔드와 백엔드 간 통신 시 10자리 숫자 ID를 전달합니다.
