# 나의 예약리스트 개발 가이드

본 문서는 고객 대시보드에서 '예약목록' 버튼을 선택했을 때 진입하는 예약 내역 확인 및 견적 확정 화면에 대한 개발 상세 내용을 담고 있습니다.

## 1. 개요
- **목적**: 사용자가 등록한 버스 예약 요청(경매) 목록을 확인하고, 기사들이 제안한 견적을 비교하여 최종 예약을 확정합니다.
- **진입 경로**: 고객 메인 대시보드 > [예약목록] 버튼 클릭

## 2. 주요 기능 및 UI 구성

### A. 예약 내역 목록 화면 (Main Screen)
- **대상**: `TB_AUCTION_REQ` 테이블에서 현재 로그인한 사용자의 요청 중 `REQ_STAT = 'BIDDING'`이고 출발일이 미래인 내역.
- **표시 항목**:
    - 여정 제목, 출발지 ↔ 도착지
    - 일정 (출발일시 ~ 도착일시)
    - 요청 인원 및 차량 정보
- **주요 버튼**:
    - **[견적리스트]**: 해당 예약 요청에 입찰한 기사들의 견적 현황 모달을 호출합니다.

### B. 견적 현황 리스트 모달 (Quotation Layout Modal)
- **기능**: 선택한 예약 건에 대해 기사들이 제출한 입찰(Bid) 정보를 리스트 형태로 보여줍니다.
- **표시 항목**: 
    - 기사 정보 (성함, 차량 사진, 별점 등)
    - 제시 금액 (천 단위 콤마 포맷팅)
    - 기사 한마디/메모
    - **본 예약 확정**: 특정 기사의 견적을 수락하여 예약을 최종 확정합니다.
        - **데이터베이스 변경 로직**:
            1. `TB_BUS_RESERVATION` 테이블: 해당 건의 `RES_STAT`를 `'CONFIRM'`으로 변경, `MOD_DT`를 현재 시간으로 업데이트.
            2. `TB_AUCTION_REQ` 테이블: 해당 요청의 `REQ_STAT`를 `'CONFIRM'`으로 변경, `MOD_DT`를 현재 시간으로 업데이트.
        - 확정 완료 후 사용자에게 알림을 표시하고 목록을 새로고침합니다.

## 3. 기술적 요구사항 (Tech Stack)
- **Frontend**: React (Hooks 기반 상태 관리), Tailwind CSS (모던 UI 디자인)
- **Backend API**:
    - `GET /api/auction/user/:userUuid`: 사용자의 예약 요청 목록 조회
    - `GET /api/auction/bids/:reqUuid`: 특정 요청에 대한 입찰 리스트 조회
    - `POST /api/auction/confirm`: 최종 예약 확정 처리

## 4. 데이터베이스 연동 테이블
- `TB_AUCTION_REQ`: 경매 마스터 정보 (상태 관리)
- `TB_AUCTION_REQ_BUS`: 요청 차량 상세
- `TB_BUS_BID`: 기사 입찰 정보 (금액 및 제안)
- `TB_DRIVER_INFO`: 기사 프로필 정보

---
> [!TIP]
> **디자인 가이드**: 대시보드의 톤앤매너를 유지하며, 견적 리스트 모달은 가독성을 위해 화이트 배경에 다크 그레이(`slate-800`) 텍스트와 프라이머리 컬러를 포인트로 사용합니다.
