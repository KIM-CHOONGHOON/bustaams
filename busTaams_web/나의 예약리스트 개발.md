# 나의 예약리스트 개발 가이드

본 문서는 고객 대시보드에서 '예약목록' 버튼을 선택했을 때 진입하는 예약 내역 확인 및 견적 확정 화면에 대한 개발 상세 내용을 담고 있습니다.

## 1. 개요
- **목적**: 사용자가 등록한 버스 예약 요청(경매) 목록을 확인하고, 기사들이 제안한 견적을 비교하여 최종 예약을 확정합니다.
- **진입 경로**: 고객 메인 대시보드 > [예약목록] 버튼 클릭

## 2. 주요 기능 및 UI 구성

### A. 예약 내역 목록 화면 (Main Screen)
- **형태**: 페이지 전환이 아닌 **오버레이 모달(Modal)** 방식으로 구현 (`App.jsx`에서 상태 관리).
- **레이아웃**: 사이드바를 제거한 **풀 와이드(Full-width)** 레이아웃으로, 여정에만 집중할 수 있는 미니멀 디자인 적용.
- **데이터 구조**: `REQ_ID` 기준으로 여러 대의 버스를 하나의 **'여정 카드'**로 그룹화하여 표시.
- **현지화**: 모든 UI 텍스트를 **한글화** 완료 (Premium Localization).
- **주요 버튼**:
    - **[여행취소]**: 해당 여정 전체(모든 버스 포함)를 취소 처리.
    - **[버스변경]**: 특정 차량의 정보를 변경. 클릭 시 기존 버스는 취소(`BUS_CHANGE`) 처리되며 즉시 **버스 재등록 모달**이 오픈됨.
    - **[버스 취소]**: 여정 내 특정 차량 한 대만 개별 취소 가능.
    - **[대시보드로 돌아가기]**: 상단 좌측에 위치한 메인 내비게이션.

### B. 견적 현황 리스트 모달 (Quotation Layout Modal)
- **UI 텍스트**: 
    - 타이틀: **"참여 기사님 현황"** (입찰 단어 제거)
    - 설명: **"참여한 기사님의 정보를 확인하세요"**
- **정렬 로직**: 최저가순이 아닌 **최신 등록순(Registration Date)**으로 정렬하여 가격 경쟁 구도 완화.
- **기사 카드 버튼 구성**:
    - **[기사정보]**: 기사님 상세 프로필 및 차량 정보 확인.
    - **[기사평가정보]**: 기존 이용객들의 리뷰 및 평점 데이터 확인.
    - **[버스취소]**: 해당 차량에 대한 선택을 취소 (붉은색 강조).
- **하단 고정 액션 바 (Sticky Footer)**:
    - **[예약취소]**: 전체 예약 요청 건을 취소 처리.
    - **[결제하기]**: 모든 구성을 확정하고 결제 단계로 진입하는 메인 버튼.

## 3. 기술적 요구사항 (Tech Stack)
- **Frontend**: React (Context/State 기반 모달 제어), Tailwind CSS (Glassmorphism & Sticky layouts)
- **Backend API**:
    - `GET /api/auction/history/:custId`: 예약 요청 목록 조회 (마스터의 `TRAVELER_CANCEL`, `BUS_CHANGE` 건은 제외, 버스의 `BUS_CANCEL`은 노출)
    - `POST /api/auction/bus-change`: 버스 변경 요청 (변경 횟수 카운트 증가, 상태값 `BUS_CHANGE` 업데이트, **기존 버스 금액 차감**)
    - `POST /api/auction/re-register-bus`: 변경된 요청에 새로운 버스 재등록 (상태 복구 `AUCTION`, **새로운 버스 총액 합산**)
    - `POST /api/auction/bus-cancel`: 개별 버스 취소 (**금액 차감**, 상태값 `BUS_CANCEL` 업데이트)
    - `GET /api/auction/bids/:reqId`: 참여 기사 리스트 조회 (최신순 정렬 적용)
    - `POST /api/auction/confirm`: 예약 확정 로직 (고도화 예정)

## 4. 데이터베이스 연동 테이블
- `TB_AUCTION_REQ`: 경매 마스터 (RES_STAT 상태 관리)
- `TB_AUCTION_REQ_VIA`: 경로 정보 (시/도 시군구 파싱용 소스)
- `TB_BUS_RESERVATION`: 기사 참여/입찰 및 예약 상태 테이블
- `TB_DRIVER_INFO / TB_BUS_DRIVER_VEHICLE`: 기사 및 차량 상세 정보

---
> [!TIP]
> **디자인 가이드**: 대시보드의 톤앤매너를 유지하며, 견적 리스트 모달은 가독성을 위해 화이트 배경에 다크 그레이(`slate-800`) 텍스트와 프라이머리 컬러를 포인트로 사용합니다.
