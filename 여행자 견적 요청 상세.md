# 여행자 견적 요청 상세 (TravelerQuoteRequestDetails) 화면 명세

본 문서는 `busTaams` 프로젝트의 **여행자 견적 요청 상세** 모달 화면에 대한 UI·데이터·API·설계 기준을 정의합니다.  
개발 환경 기준은 [`ARCHITECTURE.md`](./ARCHITECTURE.md)를 따릅니다.

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면명** | 여행자 견적 요청 상세 |
| **영문 식별자** | `TravelerQuoteRequestDetails` |
| **UI 기준 파일** | `Downloads/bustaams_web/입찰상세및수정_기사/BidDetailEdit.html` |
| **접근** | `ListOfTravelerQuotations` 모달 → 견적 카드 클릭 → `requestId` 전달 |

| **목적** | 기사가 여행자의 견적 요청 상세 내용(여정·차량·경유지·비용)을 확인하고 입찰 금액을 수정·제출 |

## 5. REST API

### 5-1. GET /api/traveler-quote-request-details

**여행자 견적 요청 상세 단건 조회**

| 항목 | 내용 |
|------|------|
| **메서드** | `GET` |
| **경로** | `/api/traveler-quote-request-details` |
| **쿼리 파라미터** | `requestId` (bigint, 필수) |
| **응답** | JSON — 마스터 + `waypoints` 배열 + `priceGuide` |
| **데이터 없을 경우** | 개발용 더미 데이터 반환 |
| **가격 가이드** | `여행자 제시 금액`: `EST_TOTAL_SERVICE_PRICE` 직접 표시 / `나의 현재 입찰가`: 입력값 |

**SQL 쿼리 (핵심)**

```sql
-- 마스터 조회
SELECT
    r.REQUEST_ID, r.USER_ID, r.BUS_TYPE, r.BUS_FUEL_EFF,
    r.PASSENGER_CNT, r.CALC_BUS_CNT, r.BUS_CNT,
    r.START_DT, r.END_DT, r.ROUND_TRIP_YN,
    r.START_ADDR, r.START_DETAIL_ADDR, r.START_LAT, r.START_LNG,
    r.END_ADDR, r.END_DETAIL_ADDR, r.END_LAT, r.END_LNG,
    r.TOTAL_DISTANCE_KM, r.TOTAL_DURATION_MIN, r.REST_AREA_CNT,
    r.FUEL_PRICE_PER_L, r.EST_FUEL_COST, r.TOTAL_TOLL_FEE,
    r.MEAL_PRICE, r.LODGING_PRICE, r.TIP_PRICE, r.INCIDENTAL_SUBTOTAL,
    r.COMMENT, r.EST_TOTAL_SERVICE_PRICE, r.STATUS, r.REG_DT
FROM TB_BID_REQUEST r
WHERE r.REQUEST_ID = ?;

-- 경유지 조회
SELECT
    w.WAYPOINT_ID, w.WAYPOINT_ADDR, w.WAYPOINT_DETAIL_ADDR,
    w.LAT, w.LNG, w.SORT_ORDER,
    w.DIST_FROM_PREV, w.TOLL_FROM_PREV, w.DURATION_FROM_PREV
FROM TB_BID_WAYPOINT w
WHERE w.REQUEST_ID = ?
ORDER BY w.SORT_ORDER;
```

## 10. DB 마이그레이션

`server.js` — `ensureBidTables(connection)` 함수에서 `CREATE TABLE IF NOT EXISTS` 로 자동 생성.  
`startServer()` 호출 시 1회 실행. **멱등성(idempotent) 보장**.

```javascript
await ensureBidTables(connection);
// ✅ TB_BID_REQUEST / TB_BID_WAYPOINT 확인 완료
```

## 13. 관련 테이블 DDL

### 13-1. TB_BID — 기사 실시간 입찰 및 제안서

```sql
-- [12] 기사 실시간 입찰 및 제안서 테이블
CREATE TABLE TB_BID (
    BID_UUID                 BINARY(16)    NOT NULL PRIMARY KEY                COMMENT '입찰 고유 식별자 (UUID)',
    REQ_UUID                 BINARY(16)    NOT NULL                            COMMENT '견적 요청 UUID (TB_BID_REQUEST.REQUEST_ID 참조)',
    DRIVER_UUID              BINARY(16)    NOT NULL                            COMMENT '입찰 기사 UUID (TB_USER.USER_UUID 참조)',
    BUS_UUID                 BINARY(16)    NOT NULL                            COMMENT '입찰 차량 UUID (TB_DRIVER_BUS.BUS_UUID 참조)',
    BASE_FARE                DECIMAL(18,0) DEFAULT 0                           COMMENT '기본 운임 (차량 운행 기본료)',
    TOLL_FEE                 DECIMAL(18,0) DEFAULT 0                           COMMENT '통행료 (고속도로 요금 합계)',
    FUEL_FEE                 DECIMAL(18,0) DEFAULT 0                           COMMENT '유류비 (연료 소비 예상 금액)',
    ROOM_BOARD_FEE           DECIMAL(18,0) DEFAULT 0                           COMMENT '숙박비 (기사 숙식 비용)',
    DRIVER_TIP               DECIMAL(18,0) DEFAULT 0                           COMMENT '기사 팁 (선택 항목)',
    TOTAL_BID_AMT            DECIMAL(18,0) NOT NULL                            COMMENT '최종 입찰 총액 (BASE_FARE + 부대비용 합계)',
    SERVICE_MEMO             TEXT                                               COMMENT '서비스 메모 / 제안서 내용 (기사 자유 입력)',
    -- ── 기사 입찰 응대 상태 ──────────────────────────────────────────────────
    BID_STAT                 ENUM(
                                 'SUBMITTED',        -- 입찰 제출됨 (기사가 견적 제출, 초기값)
                                 'RESPONDED',        -- 입찰 응대 (기사가 여행자 문의에 응대함)
                                 'RESP_CANCELLED',   -- 입찰 응대 취소 (기사가 응대 후 취소)
                                 'RESP_COMPLETED',   -- 입찰 응대 정상 종료 (기사 측 정상 완료)
                                 'SELECTED',         -- 낙찰됨 (여행자가 해당 기사를 최종 선택)
                                 'FAILED'            -- 미선택 (입찰 기간 종료 후 미낙찰)
                             ) DEFAULT 'SUBMITTED'                             COMMENT '기사 입찰 응대 상태',
    RESPOND_DT               DATETIME      DEFAULT NULL                        COMMENT '입찰 응대 일시 (RESPONDED 전환 시점)',
    RESP_CANCEL_DT           DATETIME      DEFAULT NULL                        COMMENT '입찰 응대 취소 일시 (RESP_CANCELLED 전환 시점)',
    RESP_COMPLETE_DT         DATETIME      DEFAULT NULL                        COMMENT '입찰 응대 정상 종료 일시 (RESP_COMPLETED 전환 시점)',
    -- ── 여행자 취소 관리 ──────────────────────────────────────────────────
    TRAVELER_QUOTE_CANCEL_YN  CHAR(1)      DEFAULT 'N'                         COMMENT '여행자 견적 취소 여부 (Y: 취소, N: 정상) — 여행자가 해당 입찰 견적 자체를 취소',
    TRAVELER_DRIVER_CANCEL_YN CHAR(1)      DEFAULT 'N'                         COMMENT '여행자 버스기사 선택 취소 여부 (Y: 취소, N: 정상) — 낙찰 후 여행자가 기사 선택을 취소',
    -- ── 공통 이력 ──────────────────────────────────────────────────────────
    REG_DT                   DATETIME      DEFAULT CURRENT_TIMESTAMP           COMMENT '등록 일시',
    REG_ID                   VARCHAR(30)                                       COMMENT '등록자 ID',
    MOD_DT                   DATETIME      DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '수정 일시',
    MOD_ID                   VARCHAR(30)                                       COMMENT '수정자 ID',
    CONSTRAINT FK_BID_REQ    FOREIGN KEY (REQ_UUID)    REFERENCES TB_BID_REQUEST (REQUEST_ID),
    CONSTRAINT FK_BID_DRIVER FOREIGN KEY (DRIVER_UUID) REFERENCES TB_USER (USER_UUID),
    CONSTRAINT FK_BID_BUS    FOREIGN KEY (BUS_UUID)    REFERENCES TB_DRIVER_BUS (BUS_UUID)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='기사 실시간 입찰 제안서';
```

#### BID_STAT 상태 전환 흐름

```
[기사 측]
SUBMITTED ──→ RESPONDED ──→ RESP_COMPLETED  (정상 종료)
              └──→ RESP_CANCELLED            (응대 후 취소)
SUBMITTED ──→ SELECTED                      (여행자 낙찰 선택)
SUBMITTED ──→ FAILED                        (기간 종료 후 미선택)

[여행자 측 — 별도 컬럼 관리]
TRAVELER_QUOTE_CANCEL_YN  = 'Y'  → 여행자가 해당 기사의 견적 자체를 취소
TRAVELER_DRIVER_CANCEL_YN = 'Y'  → 여행자가 낙찰 후 기사 선택을 취소 (SELECTED 상태에서 철회)
```

> **비고**: `FK_BID_REQ`는 `TB_BID_REQUEST(REQUEST_ID BIGINT)` 를 참조합니다.  
> 기존 `TB_AUCTION_REQ(REQ_UUID BINARY)` 에서 변경됐으므로, `REQ_UUID BINARY(16)` 컬럼 타입을  
> `REQUEST_ID BIGINT` 로 변경하는 별도 마이그레이션이 필요합니다.

### 13-3. 테이블 관계 요약

```
TB_BID_REQUEST (견적 요청)
    │  1:N
    └─ TB_BID_WAYPOINT (경유지)
    │  1:N
    └─ TB_BID (기사 입찰 제안서)
          │  낙찰 시
          └─ TB_BUS_RESERVATION (예약 확정 마스터)
```

| 테이블 | 역할 | 상태 컬럼 | 상태값 |
|------|------|---------|------|
| `TB_BID_REQUEST` | 여행자 역경매 요청 | `STATUS` | `OPEN` / `CLOSED` |
| `TB_BID_WAYPOINT` | 경유지 상세 | — | — |
| `TB_BID` | 기사 입찰 제안서 | `BID_STAT` | `SUBMITTED` / `SELECTED` / `FAILED` |
| `TB_BUS_RESERVATION` | 낙찰 후 예약 확정 | `RES_STAT` | `REQ` / `CONFIRM` / `DONE` / `CANCEL` |
