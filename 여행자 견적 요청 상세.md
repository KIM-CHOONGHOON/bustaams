# 여행자 견적 요청 상세 (TravelerQuoteRequestDetails) 화면 명세

본 문서는 `busTaams` 프로젝트의 **여행자 견적 요청 상세** 모달 화면에 대한 UI·데이터·API·설계 기준을 정의합니다.  
개발 환경 기준은 [`ARCHITECTURE.md`](./ARCHITECTURE.md)를 따릅니다.

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면명** | 여행자 견적 요청 상세 |
| **영문 식별자** | `TravelerQuoteRequestDetails` |

## 5. REST API

### 5-1. GET /api/traveler-quote-request-details

**여행자 견적 요청 상세 단건 조회**

| 항목 | 내용 |
|------|------|
| **메서드** | `GET` |
| **경로** | `/api/traveler-quote-request-details` |
| **쿼리 파라미터** | `reqUuid` (UUID string, 필수) / `driverUuid` (string, 선택 — 제공 시 기사 입찰가 포함) |
| **응답** | JSON — 마스터 + `waypoints` 배열 + `priceGuide` + 예약 상태 |
| **데이터 없을 경우** | 개발용 더미 데이터 반환. `driverUuid` 제공 시 `TB_BUS_RESERVATION`을 별도 조회하여 실제 `driverBiddingPrice` / `resUuid` / `resStat` / `prevBidPrice` 값을 더미 데이터에 반영 |
| **총 주행 예상 거리·시간** | `TB_BID_REQUEST.TOTAL_DISTANCE_KM` / `TOTAL_DURATION_MIN` 컬럼값 직접 출력 |
| **나의 현재 입찰가** | `TB_BUS_RESERVATION.DRIVER_BIDDING_PRICE` — 최신 회차(`ORDER BY BID_SEQ DESC LIMIT 1`) 기준 |
| **입찰 회차** | `TB_BUS_RESERVATION.BID_SEQ` — 1부터 시작, 취소 후 재입찰 시 +1 자동 증가 |
| **이전 입찰가** | `prevBidPrice` — 최근 `CANCELLATION_OF_BID` 레코드의 `DRIVER_BIDDING_PRICE`(`BID_SEQ DESC`) |
| **최종 입찰 가격(총합)** | `TB_BUS_RESERVATION.DRIVER_BIDDING_PRICE` 초기값 표시 — `RES_STAT = 'REQ'`일 때만 수정 허용, 그 외 `readOnly` |
| **CANCELLATION_OF_BID 처리** | 해당 상태 감지 시 → 프론트에서 `resStat='REQ'`로 전환, `resUuid=null` 초기화 → 신규 입찰 등록 모드로 전환 |
| **가격 가이드** | `여행자 제시 금액`: `EST_TOTAL_SERVICE_PRICE` / `나의 현재 입찰가`: `driverBiddingPrice` 초기값 |

**SQL 쿼리 (핵심)**

```sql
-- 1. TB_AUCTION_REQ 마스터 조회
SELECT
    BIN_TO_UUID(r.REQ_UUID)      AS reqUuid,
    BIN_TO_UUID(r.TRAVELER_UUID) AS travelerUuid,
    r.TRIP_TITLE                 AS tripTitle,
    r.START_ADDR, r.END_ADDR,
    r.START_DT, r.END_DT,
    r.PASSENGER_CNT              AS passengerCnt,
    r.REQ_STAT                   AS reqStat,
    COALESCE(r.REQ_AMT, 0)       AS estTotalServicePrice,
    r.EXPIRE_DT, r.REG_DT
FROM TB_AUCTION_REQ r
WHERE r.REQ_UUID = UUID_TO_BIN(?);   -- reqUuid

-- 2. TB_AUCTION_REQ_BUS 차량 조회
SELECT BUS_TYPE_CD AS busType, REQ_BUS_CNT AS busCnt
FROM TB_AUCTION_REQ_BUS
WHERE REQ_UUID = UUID_TO_BIN(?);

-- 3. TB_AUCTION_REQ_VIA 경유지 조회
SELECT VIA_ORD AS sortOrder, VIA_ADDR AS waypointAddr, STOP_TIME_MIN AS stopTimeMin
FROM TB_AUCTION_REQ_VIA
WHERE REQ_UUID = UUID_TO_BIN(?)
ORDER BY VIA_ORD;

-- 4. TB_BUS_RESERVATION — 기사 입찰 정보 (최신 회차 기준)
SELECT BIN_TO_UUID(RES_UUID) AS resUuid,
       COALESCE(DRIVER_BIDDING_PRICE, 0) AS driverBiddingPrice,
       COALESCE(RES_STAT, 'REQ') AS resStat,
       COALESCE(BID_SEQ, 1) AS bidSeq
FROM TB_BUS_RESERVATION
WHERE REQ_UUID = UUID_TO_BIN(?)     -- reqUuid
  AND DRIVER_UUID = UUID_TO_BIN(?)  -- driverUuid
ORDER BY BID_SEQ DESC LIMIT 1;

-- 5. 이전 취소 입찰가 조회 (CANCELLATION_OF_BID 중 최신)
SELECT COALESCE(DRIVER_BIDDING_PRICE, 0) AS prevBidPrice
FROM TB_BUS_RESERVATION
WHERE REQ_UUID = UUID_TO_BIN(?) AND DRIVER_UUID = UUID_TO_BIN(?)
  AND RES_STAT = 'CANCELLATION_OF_BID'
ORDER BY BID_SEQ DESC LIMIT 1;
```

### 5-2. PUT /api/traveler-quote-request-details/bid

**기사 입찰가 수정**

| 항목 | 내용 |
|------|------|
| **메서드** | `PUT` |
| **경로** | `/api/traveler-quote-request-details/bid` |
| **조건** | `TB_BUS_RESERVATION.RES_STAT = 'REQ'` 일 때만 수정 허용 |
| **수정 컬럼** | `TB_BUS_RESERVATION.DRIVER_BIDDING_PRICE` |

**요청 Body**

```json
{
  "resUuid":    "string (선택 — resUuid 직접 지정 시)",
  "requestId":  "number (선택 — resUuid 없을 때 requestId + driverUuid로 조회)",
  "driverUuid": "string (선택 — requestId와 함께 사용)",
  "bidPrice":   "number (필수 — 최종 입찰 가격)"
}
```

> `resUuid` 또는 `(requestId + driverUuid)` 중 하나 필수

**응답**

| HTTP | 내용 |
|------|------|
| `200` (수정) | `{ success: true, message: "입찰가가 성공적으로 수정되었습니다.", isNew: false }` |
| `200` (신규) | `{ success: true, message: "입찰가가 성공적으로 등록되었습니다.", isNew: true, resUuid, bidSeq }` |
| `400` | 파라미터 누락 |
| `404` | 예약 정보 없음 |
| `409` | RES_STAT ≠ 'REQ' → `{ error: "예약 상태(확정)가 '요청(REQ)' 상태가 아니므로 수정 불가", resStat }` |
| `500` | 서버 오류 |

### 5-3. PUT /api/traveler-quote-request-details/bid-cancel

**기사 입찰 취소**

| 항목 | 내용 |
||------|------|
|| **메서드** | `PUT` |
|| **경로** | `/api/traveler-quote-request-details/bid-cancel` |
|| **조건** | `TB_BUS_RESERVATION.RES_STAT = 'REQ'` 일 때만 취소 허용 |
|| **수정 컬럼** | `TB_BUS_RESERVATION.RES_STAT` → `'CANCELLATION_OF_BID'` 로 갱신 |

**요청 Body**

```json
{
  "resUuid":    "string (선택 — resUuid 직접 지정 시)",
  "requestId":  "number (선택 — resUuid 없을 때 requestId + driverUuid로 조회)",
  "driverUuid": "string (선택 — requestId와 함께 사용)"
}
```

> `resUuid` 또는 `(requestId + driverUuid)` 중 하나 필수

**응답**

| HTTP | 내용 |
||------|------|
|| `200` | `{ success: true, message: "입찰이 취소되었습니다.", resStat: "CANCELLATION_OF_BID" }` |
|| `400` | 파라미터 누락 |
|| `404` | 예약 정보 없음 |
|| `409` | RES_STAT ≠ 'REQ' → `{ error: "<상태별 오류 메시지>", resStat }` |
|| `500` | 서버 오류 |

**상태별 오류 메시지 (409)**

| RES_STAT | 오류 메시지 |
||------|------|
|| `CONFIRM` | 본 경매는 확정되어 '입찰 취소'할 수 없습니다. |
|| `DONE` | 본 경매는 여행 완료되어 '입찰 취소'할 수 없습니다. |
|| `TRAVELER_CANCEL` | 본 경매는 여행자가 여행 확정 취소하여 '입찰 취소'할 수 없습니다. |
|| `DRIVER_CANCEL` | 본 경매는 버스 기사가 여행 확정 취소하여 '입찰 취소'할 수 없습니다. |
|| `CANCELLATION_OF_BID` | 본 경매는 버스 기사가 입찰 취소하여 '입찰 취소'할 수 없습니다. |
|| `CANCELLATION_OF_AUCTION` | 본 경매는 여행자가 경매 취소하여 '입찰 취소'할 수 없습니다. |

**UI 동작**

- `RES_STAT = 'REQ'` → 확인 팝업 표시 → 확인 시 API 호출
- `RES_STAT ≠ 'REQ'` → 상태별 오류 메시지 팝업 즉시 표시 (API 호출 없음)
- 취소 성공 시 → `resStat` 상태 `CANCELLATION_OF_BID` 로 갱신 → 입력/버튼 비활성화

### 13-1. TB_BID — 기사 실시간 입찰 및 제안서

```sql
-- [12] 기사 실시간 입찰 및 제안서 테이블
CREATE TABLE TB_BID (
    BID_UUID                 BINARY(16)    NOT NULL PRIMARY KEY                COMMENT '입찰 고유 식별자 (UUID)',
    REQ_UUID                 BINARY(16)    NOT NULL                            COMMENT '견적 요청 UUID (TB_BID_REQUEST.REQUEST_ID 참조)',
    DRIVER_UUID              BINARY(16)    NOT NULL                            COMMENT '입찰 기사 UUID (TB_USER.USER_UUID 참조)',
    BUS_UUID                 BINARY(16)    NOT NULL                            COMMENT '입찰 차량 UUID (TB_DRIVER_BUS.BUS_UUID 참조)',
    BASE_FARE                DECIMAL(18,0) DEFAULT 0                           COMMENT '기본 운임 (차량 운행 기본료)',
    TOLL_FEE                 DECIMAL(18,0) DEFAULT 0                           COMMENT '통행료 (고속도로 요금 합계)',
    FUEL_FEE                 DECIMAL(18,0) DEFAULT 0                           COMMENT '유류비 (연료 소비 예상 금액)',
    ROOM_BOARD_FEE           DECIMAL(18,0) DEFAULT 0                           COMMENT '숙박비 (기사 숙식 비용)',

    SERVICE_MEMO             TEXT                                               COMMENT '서비스 메모 / 제안서 내용 (기사 자유 입력)',
    -- ── 기사 입찰 응대 상태 ──────────────────────────────────────────────────
    BID_STAT                 ENUM(
                                 'SUBMITTED',        -- 입찰 제출됨 (기사가 견적 제출, 초기값)
                                 'RESPONDED',        -- 입찰 응대 (기사가 여행자 문의에 응대함)
                                 'RESP_CANCELLED',   -- 입찰 응대 취소 (기사가 응대 후 취소)

                                 'SELECTED',         -- 낙찰됨 (여행자가 해당 기사를 최종 선택)

    REG_DT                   DATETIME      DEFAULT CURRENT_TIMESTAMP           COMMENT '등록 일시',
    REG_ID                   VARCHAR(30)                                       COMMENT '등록자 ID',

    MOD_ID                   VARCHAR(30)                                       COMMENT '수정자 ID',
    CONSTRAINT FK_BID_REQ    FOREIGN KEY (REQ_UUID)    REFERENCES TB_BID_REQUEST (REQUEST_ID),
    CONSTRAINT FK_BID_DRIVER FOREIGN KEY (DRIVER_UUID) REFERENCES TB_USER (USER_UUID),
    CONSTRAINT FK_BID_BUS    FOREIGN KEY (BUS_UUID)    REFERENCES TB_DRIVER_BUS (BUS_UUID)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='기사 실시간 입찰 제안서';
```

#### BID_STAT 상태 전환 흐름

```
[기사 측]
SUBMITTED ──→ RESPONDED ──→ RESP_COMPLETED  (정상 종료)
              └──→ RESP_CANCELLED            (응대 후 취소)
SUBMITTED ──→ SELECTED                      (여행자 낙찰 선택)
SUBMITTED ──→ FAILED                        (기간 종료 후 미선택)

[여행자 측 — 별도 컬럼 관리]
TRAVELER_QUOTE_CANCEL_YN  = 'Y'  → 여행자가 해당 기사의 견적 자체를 취소
TRAVELER_DRIVER_CANCEL_YN = 'Y'  → 여행자가 낙찰 후 기사 선택을 취소 (SELECTED 상태에서 철회)
```

> **비고**: `FK_BID_REQ`는 `TB_BID_REQUEST(REQUEST_ID BIGINT)` 를 참조합니다.  
> 기존 `TB_AUCTION_REQ(REQ_UUID BINARY)` 에서 변경됐으므로, `REQ_UUID BINARY(16)` 컬럼 타입을  
> `REQUEST_ID BIGINT` 로 변경하는 별도 마이그레이션이 필요합니다.

```sql
    BID_SEQ              INT           NOT NULL DEFAULT 1                COMMENT '입찰 회차 (1부터 시작, 취소 후 재입찰 시 +1 자동 증가)',
    DRIVER_BIDDING_PRICE DECIMAL(18,3) NOT NULL DEFAULT 0               COMMENT '버스기사 입찰가격 (PUT /bid 으로 갱신)',
    RES_STAT             ENUM(
                             'REQ',                    -- 요청: 기사 입찰 제출, 수정 가능 상태
                             'CONFIRM',                -- 확정: 여행자가 기사 선택 완료
                             'DONE',                   -- 완료: 운행 종료
                             'TRAVELER_CANCEL',        -- 여행자 취소
                             'DRIVER_CANCEL',          -- 버스기사 취소
                             'CANCELLATION_OF_BID',    -- 입찰 취소: 기사가 입찰 자체를 취소
                             'CANCELLATION_OF_AUCTION' -- 역경매 취소: 여행자가 역경매 전체를 취소
                         ) DEFAULT 'REQ'               COMMENT '예약 상태',
    CONFIRM_DT           DATETIME      DEFAULT NULL                      COMMENT '예약 확정 일시',
    REG_DT               DATETIME      DEFAULT CURRENT_TIMESTAMP        COMMENT '등록 일시',
    REG_ID               VARCHAR(30)   DEFAULT NULL                     COMMENT '등록자 ID',
    MOD_DT               DATETIME      DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '수정 일시',
    MOD_ID               VARCHAR(30)   DEFAULT NULL                     COMMENT '수정자 ID',
    KEY FK_RES_REQUEST  (REQUEST_ID),
    KEY FK_RES_TRAVELER (TRAVELER_UUID),
    KEY FK_RES_DRIVER   (DRIVER_UUID),
    CONSTRAINT FK_RES_TRAVELER FOREIGN KEY (TRAVELER_UUID) REFERENCES TB_USER (USER_UUID),
    CONSTRAINT FK_RES_DRIVER   FOREIGN KEY (DRIVER_UUID)   REFERENCES TB_USER (USER_UUID)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='버스 예약 및 매칭 확정 정보';
```

> **변경 이력**: 이전 DDL 대비 변경 사항

### 13-3. 테이블 관계 요약

```
TB_BID_REQUEST (견적 요청)
    │  1:N
    └─ TB_BID_WAYPOINT (경유지)
    │  1:N
    └─ TB_BID (기사 입찰 제안서)
          │  낙찰 시
          └─ TB_BUS_RESERVATION (예약 확정 마스터)
```

| 테이블 | 역할 | 상태 컬럼 | 상태값 |
|------|------|---------|------|
| `TB_BID_REQUEST` | 여행자 역경매 요청 | `STATUS` | `OPEN` / `CLOSED` |
| `TB_BID_WAYPOINT` | 경유지 상세 | — | — |
| `TB_BID` | 기사 입찰 제안서 | `BID_STAT` | `SUBMITTED` / `SELECTED` / `FAILED` |
| `TB_BUS_RESERVATION` | 낙찰 후 예약 확정 | `RES_STAT` | `REQ` / `CONFIRM` / `DONE` / `CANCEL` |
