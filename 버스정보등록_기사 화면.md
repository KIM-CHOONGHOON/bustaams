# 버스 정보 등록_기사 화면 개발

## 1. 개요
기사님이 자신의 버스 차량 정보를 플랫폼에 등록할 수 있는 전용 모달(`BusInformationSetup`) 개발 내역입니다. 이는 기존에 제공된 HTML 퍼블리싱을 완벽하게 React 구조로 이관하고, 전용 데이터를 처리할 수 있도록 DB 테이블 신규 설계 및 백엔드 연동을 포함하여 구현되었습니다.

## 2. 화면 진입 흐름 (FrontEnd Flow)
1. **드라이버 모드 로그인**: 기사님 계정으로 로그인 후 `DriverDashboard` 진입.
2. **바로가기 메뉴(Quick Menu) 클릭**: 화면에 있는 10개의 버튼 중 **"버스 정보"** 버튼 클릭.
3. **BusInformationSetup 모달 오픈**: 
   - `App.jsx`에 정의된 `showBusInfoModal` 상태를 통해 모달 노출 (`fixed inset-0` 형태로 전체 화면 오버레이).
4. **차량 정보 입력**:
   - `01. 기본 정보`: 차량 번호, 모델명, 연도, 주행거리 등.
   - `02. 서비스 등급`: 프리미엄 골드, 우등 등 (클릭 시 Radio 상태 연동).
   - `03. 편의 및 사양`: 다중 체크 가능한 어메니티.
   - `04. 검사 및 서류`: 최근 정기점검일, 보험 만료 예정일 등의 일자 선택.
5. **차량 등록 완료**: 하단의 [차량 등록 완료] 버튼 클릭 시 백엔드 API 호출.

## 3. BackEnd 연동 (API & Database)
- **API Endpoint**: `POST /api/driver/bus` (`server.js`)
- **수신 파라미터**: JSON 형태 (`userUuid`, `vehicleNo`, `modelNm`, `manufactureYear`, `mileage`, `serviceClass`, `amenities`, `lastInspectDt`, `insuranceExpDt`)
- **DB 테이블 (TB_DRIVER_BUS)**:
  - 구현 시 `BusTaams_Project 테이블 설계.md` 문서 최하단에 신규 테이블 규격을 명시하였으며,
  - `server.js` 내에 **"CREATE TABLE IF NOT EXISTS TB_DRIVER_BUS"** 로직을 넣어, 서버 초기 구동 및 요청 시점에 테이블이 없다면 자동으로 생성하고 INSERT 되도록 구성했습니다.

## 4. 특이사항 및 향후 과제
- 파일 업로드 UI(사업자 등록증, 운송사업 허가증, 보험증권)는 사용자가 요청한 가이드라인("화면의 형태, 버튼 변화 없이")을 맞추기 위해, 내부 상태로 연동하지 않고 UI 형태로만 남겨두었습니다. (추후 실제 파일 업로드 로직과 확장자/용량 확인이 연결되어야 합니다.)
- 모달이 열려있을 때는 뒷 배경(Dashboard)이 스크롤 되지 않도록 `document.body.style.overflow = 'hidden'` 이 적용되도록 React `useEffect`로 구성되었습니다.
