# bustaams 회원 가입 모달 (팝업) 구조 및 데이터 연동 사전 계획

메인 홈페이지에 연동될 **회원 가입 팝업 모달**의 설계 및 데이터 정합성 검토 내역입니다.

> [!IMPORTANT]
> **이미지 파일 저장 핵심 규칙**  
> 모든 사용자 관련 이미지 파일 정보(전자 서명, 증빙 서류 등)는 **`TB_DRIVER_DOCS`** 테이블로 통합 저장되어야 합니다. (이후 데이터 처리 로직 수정 시 이 규칙을 최우선으로 적용합니다.)

---

## 1. 프론트엔드 (UI) 메뉴 연결 계획
- **위치:** `busTaams_web` 프론트엔드 환경 (`App.jsx` 내 `SignUpModal` 컴포넌트)
- **작동 방식:** 
  - 상단 탭/스위치를 통해 **[여행자 / 버스기사 / 영업사원]** 중 가입 유형을 먼저 선택합니다.
  - 선택된 유형에 따라 하단 **약관 동의 및 서명 영역**이 동적으로 변합니다.

---

## 2. 약관 동의 및 계약서 노출 로직 (체크박스 구성)

가입 시 화면 하단에 표시되는 체크박스 목록은 다음과 같이 구성됩니다.

### 2-1. 공통 약관 (모든 회원 공통)
1. **[필수] 통합 이용약관**
2. **[필수] 개인정보 수집·이용 동의**
3. **[선택] 마케팅 수신 동의**

### 2-2. 회원 유형별 추가 계약서 (가입 구분에 따른 동적 노출)
상단에서 선택한 회원 유형에 따라 아래의 계약서 항목이 추가로 나타납니다.

| 회원 가입 구분 | 추가 노출 항목 | 비고 |
| :--- | :--- | :--- |
| **여행자 (CONSUMER)** | **[필수] 여행자 가입 계약서** | |
| **버스기사 (DRIVER)** | **[필수] 여행자 가입 계약서** | (기사 전용 계약서 여부 확인 필요) |
| **영업사원 (SALES)** | **[필수] 영업 회원 계약서** | |

## 3. 데이터 저장 및 매핑 규칙 (Backend & DB)

모든 가입 정보와 이미지(서명) 데이터는 아래의 규칙에 따라 저장됩니다.

### 3-1. 테이블별 저장 원칙
- **`TB_USER`**: 기본 회원 정보만 저장 (이미지 관련 컬럼 `PROFILE_IMG_PATH`, `PROFILE_FILE_ID`는 서명 저장 용도로 사용 금지).
- **`TB_DRIVER_DOCS`**: **[핵심]** 모든 약관 동의 서명 이미지를 이 테이블에 저장.
- **`TB_USER_TERMS_HIST`**: **사용 중단**. 더 이상 데이터를 입력하지 않음.

### 3-2. `TB_DRIVER_DOCS` 상세 매핑 가이드

| 화면 항목 | `DOC_TYPE` 값 | GCS 저장 경로 (`CONSENT/{DOC_TYPE}/yyyymmdd/`) |
| :--- | :--- | :--- |
| 통합 이용약관 | `TERMS_INTEGRATED` | `CONSENT/TERMS_INTEGRATED/` |
| 개인정보 수집·이용 동의 | `PRIVACY_CONSENT` | `CONSENT/PRIVACY_CONSENT/` |
| 마케팅 수신 동의 | `MARKETING_CONSENT` | `CONSENT/MARKETING_CONSENT/` |
| 버스기사 계약서 | `DRIVER_CONTRACT` | `CONSENT/DRIVER_CONTRACT/` |
| 여행자 가입 계약서 | `TRAVELER_CONTRACT` | `CONSENT/TRAVELER_CONTRACT/` |
| 영업 회원 계약서 | `PARTNER_CONTRACT` | `CONSENT/PARTNER_CONTRACT/` |

- **`DOC_TYPE_SEQ`**: 각 `CUST_ID` 및 `DOC_TYPE` 내에서 순차적으로 증가하는 값 저장 (신규 가입 시 `1`).
- **`TB_FILE_MASTER` 미사용**: 약관 및 계약서 서명 데이터는 이 테이블을 거치지 않고 `TB_DRIVER_DOCS`에서 GCS 경로를 직접 관리.

### 3-3. 파일 및 스토리지 규격
- **Bucket**: `bustaams-secure-data` (Default)
- **파일명 규칙**: `{CUST_ID}_{DOC_TYPE}_{DOC_TYPE_SEQ}.png`
- **`ORG_FILE_NM`**: 경로와 확장자를 제외한 순수 파일명 (예: `0000000001_TERMS_INTEGRATED_1`)
- **`ORG_FILE_EXT`**: 확장자명 (예: `png`)
- **최종 경로 예시**: `CONSENT/TERMS_INTEGRATED/20260428/0000000001_TERMS_INTEGRATED_1.png`

## 4. [중요] 회원 가입 처리 트랜잭션 흐름

1. **사용자 가입 요청**: 프론트엔드에서 기본 정보 + 서명(Base64) + 동의한 약관 리스트 전송.
2. **GCS 업로드**: 서명을 각 `DOC_TYPE`별 경로에 맞춰 업로드 및 `FILE_ID` 생성.
3. **`TB_FILE_MASTER` 등록**: 생성된 `FILE_ID`와 경로 정보 기록.
4. **`TB_USER` 생성**: 회원 마스터 정보 저장.
5. **`TB_DRIVER_DOCS` 생성**: 동의한 각 약관/계약서마다 1행씩 서명 이미지 연결 정보(FILE_ID)와 함께 저장.
