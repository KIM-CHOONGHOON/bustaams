# 버스 운행 완료 목록 화면 (`BusOperationCompletionList`)

**정본 DDL·컬럼명**은 [`BusTaams 테이블.md`](./BusTaams%20테이블.md) — 특히 `TB_BUS_RESERVATION`, `TB_AUCTION_REQ`, `TB_AUCTION_REQ_BUS`, `TB_USER` 를 따른다. (구 `TRAVELER_UUID` / `DRIVER_UUID` / `RES_UUID` BINARY 스키마는 **미사용**이며, API는 `TRAVELER_ID`·`DRIVER_ID`·`RES_ID` **varchar** 기준으로 동작한다.)

---

## 0. 식별자·필드 매핑

| 구분 | DB 컬럼 | 타입(정본) | REST/JSON |
|------|---------|------------|-----------|
| 기사(예약 행) | `TB_BUS_RESERVATION.DRIVER_ID` | varchar(10) = `TB_USER.CUST_ID` | 쿼리 `driverId` / `userId` / `userUuid` 동의어 1개 — 서버가 `CUST_ID`로 해석 |
| 완료 예약 | `RES_ID` | varchar(10) | `resId` |
| 견적 마스터 | `REQ_ID` | varchar(10) | `reqId` |
| 견적 버스 | `REQ_BUS_ID` | varchar(10) | `auctionReqBuses[].reqBusId` |
| 상태 | `DATA_STAT` = `'DONE'` | enum | `dataStat` |
| 여행자(비정규) | `TRAVELER_ID` | varchar(10) | (상세에서 조인만) |
| 리뷰(참고) | `TB_TRIP_REVIEW.WRITER_ID` / `DRIVER_ID` | varchar(10) | 본 화면 API에 없음 |

**삭제한 예전 API 필드:** `resUuid`, `reqUuid`, `reqBusUuid`, `resStat`(중복) — 응답·요청에서 제거. 상태는 **`dataStat`** 만 사용.

## 1. 개요

| 항목 | 내용 |
|------|------|
| **화면** | 버스 운행 완료 목록 (모달 `z-[100]`) |
| **진입** | 기사 **MainDashBoard** 등 — 바로가기 **「버스 운행 완료 목록」** (프로젝트에 맞게 연결) |

| **정렬** | `END_DT` 내림차순 |

## 3. REST API

### 3-1. 목록

| | |
|---|---|
| **GET** | `/api/bus-operation-completion-list` |
| **구현** | `busTaams_server/routes/busOperationCompletionList.js` |

**Query**

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| `driverId` | 1개 필수* | `TB_USER.USER_ID` / `CUST_ID` / 레거시 식별 — 서버가 `resolveCustIdByUserKey`로 `DRIVER_ID`에 맞게 변환 |
| `userId` | ↑ 동의어 | `driverId`와 동일 의미 |
| `driverUuid` | ↑ 동의어 | 이전 세션 키 이름 (값은 UUID가 아닐 수 있음) |
| `from` | O | `YYYY-MM-DD` |
| `to` | O | `YYYY-MM-DD` |

\* `driverId`·`userId`·`driverUuid` **중 하나**.

**검증 오류 (400):** 날짜 형식, `to < from`, 기간 &gt; 365일 → `조회 범위는 1년 이내만 가능합니다.`

**Response 200**

```json
{
  "total": 0,
  "items": [
    {
      "resId": "0000000001",

      "endDtDate": "YYYY-MM-DD",
      "startDt": "ISODate",

```

### 3-2. 상세

| | |
|---|---|
| **GET** | `/api/bus-operation-completion-details` |
| **구현** | `busTaams_server/routes/busOperationCompletionDetails.js` |

**Query**

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| `driverId` / `userId` / `driverUuid` | 1개 | 목록과 동일 |
| `resId` | `resId` 또는 `reqId` 중 하나 | `TB_BUS_RESERVATION.RES_ID` (소문자 정규화) |
| `reqId` | 위와 택1 | `TB_AUCTION_REQ.REQ_ID` — `resId` 없을 때 |

**Response 200 (주요):** `travelerName`, `tripTitle`, `startDt`, `endDt`, `passengerCnt`, `travelerRequestKrw`, `driverBidKrw`, `auctionReqBuses[]`, `timeline[]`, `resId`, `reqId`, `dataStat`

## 4. BackEnd

- `lib/resolveCustIdByUserKey.js` — `USER_ID`/`CUST_ID` → `CUST_ID` (10자리) — `DRIVER_ID` 조회에 사용
- `server.js` — 동일 헬퍼 `require` (다른 API와 공용)

**조건 (정본):** `TB_BUS_RESERVATION`.`DATA_STAT` = `'DONE'`, `INNER JOIN TB_AUCTION_REQ` on `REQ_ID`, 기간 `DATE(r.END_DT)` between `from` and `to`, `DRIVER_ID` = 해석된 기사 `CUST_ID`.

**레거시 DB:** `ER_BAD_FIELD_ERROR` 시에만 `RES_UUID`·`RES_STAT`·`DRIVER_UUID` 등 구 쿼리 폴백 (운영은 `BusTaams` 마이그레이션 권장).

---

## 5. 문제 해결 (404)

- **Cannot GET** → Express 재기동, `registerBusOperationCompletionList(pool, app)` 등록 여부
- **빈 목록** — `END_DT` null 제외, From/To·기사 `DRIVER_ID` 일치, `INNER JOIN` 으로 `TB_AUCTION_REQ` 없으면 제외
