# 활성 입찰 섹션 (DashboardTopSection — 우측 카드)

운전기사 메인 대시보드 상단 그리드 **우측** 다크 그린 카드(**「활성 입찰」**)에 표시하는 지표와 데이터 규칙을 정의합니다.  
구현은 `DriverDashboard.jsx`의 `DashboardTopSection`(우측 열) · `GET /api/driver/dashboard` 응답 필드를 사용합니다.

---

## 1. 화면 요구사항

| 표시 제목 | 데이터 의미 | 집계 규칙 |
|-----------|-------------|-----------|
| **입찰 건수** | 입찰 진행 중인 건수 | `RES_STAT = 'REQ'` 인 행 **건수** |
| **입찰 금액** | 위 건들의 입찰가 합 | `RES_STAT = 'REQ'` 인 행의 **`DRIVER_BIDDING_PRICE` 합계** |
| **운행 예정** | 운행 확정 건수 | `RES_STAT = 'CONFIRM'` 인 행 **건수** |
| **운행 예정 금액** | 위 건들의 입찰가 합 | `RES_STAT = 'CONFIRM'` 인 행의 **`DRIVER_BIDDING_PRICE` 합계** |

공통 조건:

- `TB_BUS_RESERVATION.DRIVER_UUID` = **로그인한 기사 UUID** (`USER_UUID` 문자열을 `UUID_TO_BIN`으로 매칭)

출력 형식 예(실제 값은 DB 기준):

- 입찰 건수: `9999건` 형태(천 단위 구분)
- 입찰 금액: `₩9,999,000` 형태
- 운행 예정: `9999건`
- 운행 예정 금액: `₩9,999,000`

카드 상단에는 섹션 제목 **「활성 입찰」** 과 경매 망치 아이콘(gavel), 하단에는 얇은 구분선(장식)을 둡니다.

---

## 2. REST API (`GET /api/driver/dashboard`)

동일 엔드포인트 응답에 아래 필드가 포함됩니다.

| 필드 | 타입 | 설명 |
|------|------|------|
| `bidCount` | number | 입찰 건수 (`REQ`) |
| `bidAmountSum` | number | 입찰 금액 합 (`REQ`) |
| `confirmCount` | number | 운행 예정 건수 (`CONFIRM`) |
| `confirmAmountSum` | number | 운행 예정 금액 합 (`CONFIRM`) |
| `activeBids` | number | 하위 호환용, **`bidCount`와 동일** |

### SQL (한 번에 집계)

```sql
SELECT
  COALESCE(SUM(CASE WHEN RES_STAT = 'REQ' THEN 1 ELSE 0 END), 0) AS bid_count,
  COALESCE(SUM(CASE WHEN RES_STAT = 'REQ' THEN DRIVER_BIDDING_PRICE ELSE 0 END), 0) AS bid_amount_sum,
  COALESCE(SUM(CASE WHEN RES_STAT = 'CONFIRM' THEN 1 ELSE 0 END), 0) AS confirm_count,
  COALESCE(SUM(CASE WHEN RES_STAT = 'CONFIRM' THEN DRIVER_BIDDING_PRICE ELSE 0 END), 0) AS confirm_amount_sum
FROM TB_BUS_RESERVATION
WHERE DRIVER_UUID = UUID_TO_BIN(?);
```
