# 예약 목록 및 기사 변경 로직 요건 정의서 (v2 - 수정본)

## 1. 기사 변경 (구 버스 취소) 기능 개요
사용자가 이미 확정된 예약을 취소하는 대신, 다른 기사의 응찰을 다시 받기 위해 현재 확정을 취소하고 경매 상태로 되돌리는 기능.

## 2. 상세 비즈니스 로직

### A. 버튼 명칭 및 노출 조건
- **[기사 변경]** (구 버스 취소): 차량은 유지하고 응찰한 기사만 교체하는 기능.
    - **노출 조건**: 해당 차량(`REQ_BUS_ID`)에 대해 응찰/확정된 기사가 1:1로 매핑되어 존재해야 함 (`RES_STAT` 존재 시).
- **[버스 취소]** (구 버스 변경): 해당 차량 요청 자체를 삭제/취소하는 기능 (버스 삭제의 의미).
    - **노출 조건**: 항상 표시.

### B. 데이터베이스 변화 (트랜잭션 처리)

#### 1) TB_AUCTION_REQ (마스터 여정 테이블)
- **상태값 변경**: `DATA_STAT`을 **`'AUCTION'`**으로 변경.
- **금액 유지**: `REQ_AMT` (전체 요청 금액)을 차감하지 않고 **보존**.
- **횟수 관리**: `BUS_CHANG_CNT` 컬럼값을 **1 증가**.
- **제한 조건**: `BUS_CHANG_CNT`가 1회 이상인 경우 변경 불가 (이미 변경됨 메시지 노출).

#### 2) TB_AUCTION_REQ_BUS (요청 차량 상세 테이블)
- **상태값 변경**: `'AUCTION'`으로 변경하여 재응찰이 가능하도록 함.

#### 3) 알림톡 발송 (기사 통보)
- **대상**: 기존에 배정(확정)되었던 기사님.
- **발송 시점**: 기사 변경 트랜잭션 완료 직후.
- **메시지 내용**: `[busTaams] 예약 취소 안내\n\n기사님, 예약된 여정(요청번호: {reqId})이 고객님의 요청으로 취소되었습니다.`

### C. 버스 취소 (구 버스 변경 - 버스 삭제의 의미)
- **기능**: 여정 내 특정 차량 요청을 영구적으로 삭제.
- **데이터베이스 변화**:
    1. **잔여 대수 체크**: 해당 여정(`REQ_ID`)에 활성화된 차량 대수(상태가 취소가 아닌 건)를 확인.
    2. **TB_AUCTION_REQ (마스터)**:
        - 잔여 차량이 **1대**인 경우: 상태값을 **`'TRAVELER_CANCEL'`**로 변경 (여정 전체 취소).
        - 잔여 차량이 **2대 이상**인 경우: 상태값을 **변경하지 않음**.
        - 공통: `REQ_AMT` (전체 요청 금액)에서 해당 버스 금액만큼 **차감**.
    3. **TB_AUCTION_REQ_BUS (상세)**: 해당 차량의 상태를 **`'TRAVELER_CANCEL'`**로 변경.
- **노출 조건**: 항상 표시.

## 3. 화면 표시 데이터 (API 요건)
- 예약 목록 조회 시, 각 차량 슬롯에 연결된 예약 상태를 가져와야 함.
- 불필요한 카운트(`BID_COUNT`)는 제거하고, 1:1 매핑된 예약 데이터의 상태를 기준으로 처리.
