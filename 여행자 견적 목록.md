# 여행자 견적 목록 (ListOfTravelerQuotations) 화면 명세

본 문서는 `busTaams` 프로젝트의 **여행자 견적 목록** 모달 화면에 대한 UI·데이터·API·설계 기준을 정의합니다.  
개발 환경 기준은 [`ARCHITECTURE.md`](./ARCHITECTURE.md)를 따릅니다.

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면명** | 여행자 견적 목록 |
| **영문 식별자** | `ListOfTravelerQuotations` |
| **접근** | 운전기사 대시보드 → 바로가기 메뉴 → **「여행자 견적 목록」** |
| **형태** | 오버레이 모달 (`z-[150]`) |
| **목적** | 여행자가 버스 기사 다수에게 등록한 **역경매(Reverse Auction)** 견적 요청 목록을 기사에게 제공, 기사가 원하는 견적을 선택·입찰할 수 있도록 함 |

### 1-1. 역경매(역경매) 개념

```
여행자 (1명)
  └─ 버스 대절 견적 요청 등록 (TB_BID_REQUEST)
        ↓
기사 다수 (N명) 가 목록을 조회하고 견적을 제출
  └─ 기사가 「여행자 견적 목록」에서 **입찰 진행 중**인 항목 확인
     (`TB_AUCTION_REQ.RES_STAT = 'BIDDING'`)
  └─ 항목 클릭 → 「TravelerQuoteRequestDetails」 모달 호출 → 상세 조회 및 입찰
```

> **`TB_AUCTION_REQ.RES_STAT` ENUM (현행)**  
> `'BIDDING'`, `'CONFIRM'`, `'DONE'`, `'TRAVELER_CANCEL'`, `'DRIVER_CANCEL'`
>
> | 값 | 의미 (여행자 견적 목록) |
> |----|------------------------|
> | `BIDDING` | 역경매·입찰 진행 중 — **목록에 표시** |
> | `CONFIRM` | 확정(예: 기사·여행자 합의 등) — 목록 **미**표시 |
> | `DONE` | 운행·거래 종료 — 목록 미표시 |
> | `TRAVELER_CANCEL` | 여행자 취소 — 목록 미표시 |
> | `DRIVER_CANCEL` | 기사 취소 — 목록 미표시 |
>
> **구 스키마 참고 (`REQ_STAT` + `CLOSED` / `CANCELED` 사용 시)**  
> DB가 `REQ_STAT`·`CLOSED`·`CANCELED`를 쓰던 시기와 비교하면, 업무적으로는 `RES_STAT = 'BIDDING'`이 역경매 진행에 해당하고, 마감·취소는 `DONE`·`TRAVELER_CANCEL`·`DRIVER_CANCEL` 등으로 세분화된 것으로 보면 된다. (마이그레이션 후 컬럼명은 **`RES_STAT`** 로 통일.)

## 5. 데이터 모델 (DB 테이블)

### 5-1. TB_AUCTION_REQ — 역경매 견적 요청 메인 (조회 대상)

| 컬럼 | 타입 | 목록 표시 |
|------|------|---------|
| `REQ_UUID` | `BINARY(16)` PK | 카드 클릭 시 `TravelerQuoteRequestDetails`로 전달 (`reqUuid`) |
| `TRAVELER_UUID` | `BINARY(16)` | — |
| `TRIP_TITLE` | `VARCHAR(255)` | **여정 제목** |
| `START_ADDR` | `VARCHAR(255)` | **출발지** |
| `END_ADDR` | `VARCHAR(255)` | **도착지** |
| `START_DT` | `DATETIME` | **출발 일시** |
| `END_DT` | `DATETIME` | **도착 일시** |
| `PASSENGER_CNT` | `INT` | **탑승 인원** |
| `RES_STAT` | `ENUM('BIDDING','CONFIRM','DONE','TRAVELER_CANCEL','DRIVER_CANCEL')` | 조회 조건: **`'BIDDING'`** 만 목록에 포함 |
| `REQ_AMT` | `DECIMAL` | **여행자 제시 금액** (`estTotalServicePrice` 로 응답) |
| `EXPIRE_DT` | `DATETIME` | **입찰 마감 일시** |
| `REG_DT` | `DATETIME` | **등록 일시** |

### 5-2. TB_AUCTION_REQ_BUS — 요청 차량 상세 (차량 종류·대수)

| 컬럼 | 역할 |
|------|------|
| `REQ_BUS_UUID` | PK |
| `REQ_UUID` | `TB_AUCTION_REQ.REQ_UUID` 와 JOIN |
| `BUS_TYPE_CD` | 차량 유형 → `busType` 으로 응답 |
| `REQ_BUS_CNT` | 요청 대수 → `busCnt` 으로 응답 |

### 5-3. TB_AUCTION_REQ_VIA — 경유지 상세 (건수만 집계)

| 컬럼 | 역할 |
|------|------|
| `VIA_UUID` | PK |
| `REQ_UUID` | `TB_AUCTION_REQ.REQ_UUID` 와 JOIN (COUNT 집계 → `waypointCount`) |
| `VIA_ORD` | 방문 순서 |
| `VIA_ADDR` | 경유지 장소명 |

### 5-4. 목록 조회 대상 (업무 명세)

아래는 **여행자 견적 목록**에 노출되어야 하는 `TB_AUCTION_REQ` 행에 대한 업무 조건이다.

| 조건 | 설명 |
|------|------|
| **상태** | **`RES_STAT = 'BIDDING'`** — 역경매·입찰이 진행 중인 요청만 대상으로 한다. |
| **출발 일자** | **시스템(서버) 기준 오늘(`CURDATE()`)보다 뒤인 날**에 출발하는 요청만 대상으로 한다. 즉 **`DATE(START_DT) > CURDATE()`** — **당일(오늘) 출발은 제외**, 익일 0시부터 해당. 비교에는 **`START_DT`**의 **날짜 부분**을 사용한다. |

**`START_ADDR` 관련 보정:** 요청 문구에 `START_ADDR`(출발지 주소)가 나올 수 있으나, 주소 문자열은 날짜와 직접 비교할 수 없다. 동일 의도는 **`START_DT`** 로 **`DATE(START_DT) > CURDATE()`** 를 적용하는 것이 맞다.

> **구현 참고:** [§8-1](#8-1-get-apilist-of-traveler-quotations) API는 DB 컬럼 **`REQ_STAT`**(문서상 `RES_STAT`와 동일 의미) 및 **`DATE(r.START_DT) > CURDATE()`** 를 적용한다. 동일 출발일 제외는 §8-1과 같다.

### 5-5. 현재 목록 쿼리 대비 변경점 (요약)

`GET /api/list-of-traveler-quotations`의 `WHERE` 절은 업무 명세(§5-4)와 아래처럼 맞춘다.

| 구분 | 현재 구현 | 업무 명세(본 문서 §5-4) |
|------|-----------|---------------------------|
| 입찰 진행 여부 | `r.REQ_STAT = 'BIDDING'` | 동일 (DB 컬럼명; 문서 `RES_STAT`) |
| 출발 일자 | **`DATE(r.START_DT) > CURDATE()`** | 동일 — **당일 제외**, `CURDATE()`는 MySQL **서버 기준 오늘 날짜** |
| 기사별 동일 출발일 제외 | `driverUuid` 있을 때 `NOT EXISTS(...)` | 동일 |

**`>=` vs `>`:** `DATE(START_DT) >= CURDATE()` 는 **오늘·내일·…** 모두 포함한다. 시스템 일자 **이후**만 보려면 **`DATE(START_DT) > CURDATE()`** 가 맞다.

## 8. REST API

### 8-1. GET /api/list-of-traveler-quotations

**역경매 견적 요청 목록 조회 — `TB_AUCTION_REQ.REQ_STAT = 'BIDDING'`** + **`DATE(START_DT) > CURDATE()`**  
(`CURDATE()` = MySQL 서버 **시스템 일자**(오늘). 당일 출발 건은 목록에서 제외.)

| 항목 | 내용 |
|------|------|
| **메서드** | `GET` |
| **경로** | `/api/list-of-traveler-quotations` |
| **쿼리** | `driverUuid` (선택) — 기사 로그인 시 전달 권장 |
| **응답** | JSON — `total` + `items` 배열 |
| **데이터 없을 경우** | 개발용 더미 데이터 반환 |

**동일 출발일 제외 (`driverUuid`가 있을 때)**  
로그인 기사(`TB_BUS_RESERVATION.DRIVER_UUID`)에 대해, **다른 견적**(`ar.REQ_UUID <> r.REQ_UUID`)에 **`TB_BUS_RESERVATION.RES_STAT = 'CONFIRM'`** 인 예약이 있고, 그 견적의 `TB_AUCTION_REQ.START_DT` **날짜**가 `r.START_DT`와 같으면 해당 `r` 행은 목록에서 제외한다. (`TB_AUCTION_REQ.RES_STAT`와 이름이 같아도 **예약 테이블**의 `RES_STAT`이며, ENUM 구성은 `TB_AUCTION_REQ`와 다를 수 있다.)

**SQL 쿼리 (핵심)**

```sql
SELECT ...
FROM TB_AUCTION_REQ r
LEFT JOIN TB_AUCTION_REQ_BUS b ON b.REQ_UUID = r.REQ_UUID
LEFT JOIN TB_AUCTION_REQ_VIA v ON v.REQ_UUID = r.REQ_UUID
WHERE r.REQ_STAT = 'BIDDING'
  AND DATE(r.START_DT) > CURDATE()
  AND NOT EXISTS (
    SELECT 1
      FROM TB_BUS_RESERVATION res
      INNER JOIN TB_AUCTION_REQ ar ON ar.REQ_UUID = res.REQ_UUID
     WHERE res.DRIVER_UUID = UUID_TO_BIN(?)
       AND res.RES_STAT = 'CONFIRM'
       AND DATE(ar.START_DT) = DATE(r.START_DT)
       AND ar.REQ_UUID <> r.REQ_UUID
  )
GROUP BY ...
ORDER BY r.REG_DT DESC;
```

**응답 스키마**

```json
{
  "total": "number",
  "items": [
    {
      "reqUuid":              "string (UUID)",
      "tripTitle":            "string | null",
      "busType":              "string | null",
      "busCnt":               "number",
      "passengerCnt":         "number",
      "startDt":              "datetime",
      "endDt":                "datetime",
      "startAddr":            "string",
      "endAddr":              "string",
      "estTotalServicePrice": "number",
      "reqStat":              "string (예: BIDDING — DB는 TB_AUCTION_REQ.RES_STAT)",
      "expireDt":             "datetime",
      "regDt":                "datetime",
      "waypointCount":        "number"
    }
  ]
}
```

**응답 필드 `reqStat`:** 스키마상 소스는 **`TB_AUCTION_REQ.RES_STAT`**. 레거시 API는 컬럼·별칭을 `REQ_STAT` / `reqStat`로 둔 경우가 있어, 백엔드 마이그레이션 시 `RES_STAT`와 맞추면 된다.

## 9. 프론트엔드 연동 흐름

```
App.jsx
  └─ showQuotationModal (useState)
       ├─ DriverDashboard → QuickMenu → '여행자 견적 목록' 클릭
       │     → setShowQuotationModal(true)
       └─ {showQuotationModal &&
             <ListOfTravelerQuotations close={...} currentUser={user} />}
               │
               ├─ useEffect on mount
               │     → fetch GET /api/list-of-traveler-quotations?driverUuid=
               │     → setItems(result.items)  ← 견적요청 RES_STAT=BIDDING + 기사별 동일출발일 제외(예약 RES_STAT=CONFIRM)
               │
               └─ 카드 클릭 → setSelectedReqUuid(item.reqUuid)   ← UUID 전달
                     └─ {selectedReqUuid &&
                           <TravelerQuoteRequestDetails
                             reqUuid={selectedReqUuid}
                             close={() => setSelectedReqUuid(null)}
                             currentUser={currentUser}
                           />}
```
