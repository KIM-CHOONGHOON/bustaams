# 견적리스트_기사 화면 개발

## 1. 개요
기사님이 소비자의 버스 예약 요청 내역을 조회하고 해당 요청에 대해 실시간 견적 현황을 파악한 뒤, 견적을 제안할 수 있도록 지원하는 `QuotationRequests` 모달/화면 개발 내역입니다. 기존에 제공된 HTML 디자인의 CSS 및 UI를 최대한 동일하게 유지하되, 데이터 연동과 React 상태 관리를 추가하여 기능적인 대시보드 화면으로 연계했습니다.

## 2. 화면 진입 흐름 및 주요 기능
1. **드라이버 모드 로그인**: 기사님 계정으로 접속 후 `DriverDashboard` 화면 진입.
2. **견적 리스트 메뉴 클릭**: 대시보드 화면 중앙의 `바로가기 메뉴(Quick Menu)`에서 '견적 리스트' 클릭.
3. **QuotationRequests 전체 화면 모달 오픈**: 
   - `App.jsx`의 `showQuotationModal` 상태가 `true`로 변경되며, 팝업(전체 덮기) 형태로 화면 표출.
   - 열린 상태에서는 배경 스크롤이 차단(`document.body.style.overflow = 'hidden'`)됩니다.
4. **견적 예약 정보 조회 (Data Fetching)**:
   - 오픈과 동시에 `GET /api/driver/quotation-requests` 백엔드 API를 호출해 최근 예약 요청 및 경쟁 견적 현황 데이터를 가져옵니다.
   - 받아온 JSON 데이터를 기반으로, 예약 요약 정보(출발지/도착지 및 인원수)와 입찰에 참여한 내역(버스 종류, 연식, 편의 시설 등)들의 카드 리스트를 동적으로 렌더링합니다.
5. **화면 종료(Back Navigation)**:
   - 화면 좌측 상단의 '대시보드 돌아가기' 버튼을 클릭하면 모달 상태가 해제되며 기존 화면으로 복귀합니다.

## 3. BackEnd 연동 및 DB 상세 (TB_BUS_REQUEST, TB_BID)
- **API Endpoint**: `GET /api/driver/quotation-requests` (`server.js`)
- **수단 및 목적**: 현재는 UI 구성요소를 정확히 매핑하기 위해 Dummy Data를 세팅하여 `json` 포맷의 배열 객체를 직접 반환하여 화면 렌더링용으로 구성했습니다.
- **DB 스키마 추가 사항**:
  프로젝트의 통합 DB 관리 문서인 `BusTaams_Project 테이블 설계.md` 에 다음 두 테이블을 정식 명세로 추가했습니다.
  - **`TB_BUS_REQUEST`**: 소비자가 입력한 예약 요청 정보(출발/도착, 인원수, 차량등급 등)를 관리하는 테이블입니다.
  - **`TB_BID`**: 개별 예약 요청 건에 대해 여러 기사님들이 투찰(견적 제출)하는 정보(견적가, 사용 차량, 입찰 상태 등)를 추적 관리하는 테이블입니다.

## 4. 특기 사항 및 디자인 고려 요소
- **소비자 측 UI 용어 조정 관점**: HTML 스크립트는 본래 "(특정 기사님이) OOO 견적을 제안했습니다"와 같이 소비자가 기사님을 고르는 형태를 띠었으나, 사용자님의 지시사항을 반영하여 기사님 측면에서 소비자의 **예약 요청 내역**을 보고 타 기사의 투찰정보도 함께 둘러보며 "견적 내용/선택"을 판가름할 수 있는 종합 뷰로 해석 및 연동했습니다. (원본 UI의 "모양과 구성, 색상"은 변형 없이 동일하게 가져왔습니다)
