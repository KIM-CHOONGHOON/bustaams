# 실시간 입찰 기회 섹션 (AuctionList Section) 화면 명세

본 문서는 `busTaams` 프로젝트의 **운전기사 메인 대시보드(DriverDashboard)** 내  
**실시간 입찰 기회** 섹션에 대한 UI·데이터·API·설계 기준을 정의합니다.  
개발 환경 기준은 [`ARCHITECTURE.md`](./ARCHITECTURE.md)를 따릅니다.

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면명** | 실시간 입찰 기회 섹션 |
| **영문 식별자** | `AuctionList` |
| **위치** | `DriverDashboard` 페이지 내 하단 섹션 (`lg:col-span-2`) |
| **형태** | 페이지 내 섹션 (독립 페이지/모달 아님) |
| **목적** | 역경매 진행 중(`REQ_STAT = 'BIDDING'`)인 여행자 견적 요청 목록을 기사에게 노출하고 입찰 참여 또는 입찰 가격 변경을 유도 |

### 1-1. 목록 건수와 순환

- 조회 결과가 **0건**이면 카드 대신 안내 문구만 표시한다 ([4-1](#4-1-목록-표시-개수-및-스크롤)).
- **1건**일 때: 해당 카드 **1줄(1건)** 만 표시하고, **자동 순환 타이머는 동작하지 않는다.**
- **2건 이상**일 때: **항상 1건씩만** 보이고, **5초마다** 다음 항목으로 바뀌며, 전환 시 **위 방향(아래→위) 슬라이드** 애니메이션으로 표시한다 (순서는 `REG_DT DESC` 기준 배열에서 인덱스 순환).

## 3. UI 레이아웃

### 3-1. 썸네일(버스 사진) 비표시

- 카드 **왼쪽에 버스·차량 사진을 넣지 않는다.** (`<img>` 썸네일 영역 없음)
- **표시 항목**(상→하): 제목 + **요청 상태 배지**(`REQ_STAT`, 예: `BIDDING`) · 우측 **등록 일시**(`REG_DT`) · **노선**(출발지 → 도착지) · 2×2 그리드(출발·도착 일시, 탑승 인원, 차량 유형×대수) · 하단 **예산 총액**(`REQ_AMT`) · **입찰 참여 / 입찰 제시 변경** 및 우측 chevron(`>`).

### 3-2. 레이아웃 다이어그램 (카드 1건 예시)

```

│  │  ┌─────────────────────┬─────────────────────┐                      │ │
```

## 4. 표시 규칙

### 4-1. 목록 표시 개수 및 스크롤

| 항목 | 규칙 |
|------|------|
| **동시 표시 개수** | **1건** (카드 1줄) |
| **자동 전환** | `items.length >= 2` 일 때만 `setInterval(5000ms)` 로 `currentIndex` 를 `(i + 1) % length` 로 갱신 |
| **전환 연출** | 새 카드가 아래에서 위로 들어오는 느낌의 짧은 슬라이드 애니메이션 (`animate-auction-slide-up`) |
| **데이터 정렬** | API `ORDER BY r.REG_DT DESC` — 프론트는 응답 배열 순서대로 인덱스 순환 |
| **1건일 때** | 타이머 없음 · 해당 1건만 표시 |
| **0건일 때** | 카드 영역에 **「등록된 입찰이 없습니다」** 문구 표시 (백엔드 `emptyMessage` 와 동기화 가능) |

### 4-2. 조회 조건

- `TB_AUCTION_REQ.REQ_STAT = 'BIDDING'` 인 항목만 표시
- **현행 DB**: 위 조건에 해당하는 데이터는 **1건** 존재
- 기사 본인이 이미 입찰한 항목도 포함하여 표시 (버튼 텍스트로 구분)

### 4-3. 카드에 표시하지 않는 요소

| 항목 | 규칙 |
|------|------|
| **버스·차량 썸네일 이미지** | **출력하지 않음** (URL·`<img>`·플레이스홀더 모두 사용하지 않음) |
| **헤더 필터 아이콘** | 섹션 제목만 표시 (우측 필터 버튼 없음) |
| 그 외 | [3-1](#3-1-썸네일버스-사진-비표시) 의 정보 행만 출력 |

## 5. 데이터 모델 (DB 테이블)

### 5-1. TB_AUCTION_REQ — 역경매 견적 요청 (목록 조회)

| 컬럼 | 타입 | 표시 용도 |
|------|------|---------|
| `REQ_UUID` | `BINARY(16)` PK | 입찰 참여 시 식별자 전달 |
| `TRAVELER_UUID` | `BINARY(16)` | `TB_BUS_RESERVATION` 매핑 기준 |
| `TRIP_TITLE` | `VARCHAR(255)` | 카드 **제목** |
| `START_ADDR` | `VARCHAR(255)` | 노선 표시 (출발지) |
| `END_ADDR` | `VARCHAR(255)` | 노선 표시 (도착지) |
| `START_DT` | `DATETIME` | 카드 **출발** 일시 |
| `END_DT` | `DATETIME` | 카드 **도착** 일시 |
| `PASSENGER_CNT` | `INT` | 카드 **탑승** 인원 |
| `REQ_STAT` | `ENUM('BIDDING','CLOSED','CANCELED')` | 조회 조건: `'BIDDING'` 만 · 카드 **상태 배지** 텍스트 |
| `REQ_AMT` | `DECIMAL` | 카드 **예산 총액** 표시 |
| `EXPIRE_DT` | `DATETIME` | (예약) 향후 마감 관련 기능용 — 현재 카드 배지에는 미사용 |
| `REG_DT` | `DATETIME` | 카드 우측 **등록** 일시 · 목록 정렬 기준 |

### 5-2. TB_AUCTION_REQ_BUS — 요청 차량 상세

| 컬럼 | 타입 | 표시 용도 |
|------|------|---------|
| `REQ_UUID` | `BINARY(16)` FK | JOIN 조건 |
| `BUS_TYPE_CD` | `VARCHAR(30)` | 차량 종류 표시 (예: `STANDARD_45`) |
| `REQ_BUS_CNT` | `INT` | 요청 차량 대수 — 목록 API에서 **동일 요청 행 합산**(`SUM`) 후 표시 |

### 5-3. TB_BUS_RESERVATION — 기사 입찰 예약 (버튼 텍스트 판단)

| 컬럼 | 타입 | 역할 |
|------|------|------|
| `REQ_UUID` | `BINARY(16)` | `TB_AUCTION_REQ.REQ_UUID` 매핑 |
| `DRIVER_UUID` | `BINARY(16)` | 현재 로그인 기사 UUID로 필터 |
| `BID_SEQ` | `INT` | 입찰 회차 — **최대값(MAX) ROW** 기준으로 판단 |
| `RES_STAT` | `ENUM(...)` | 버튼 텍스트 결정 기준 |

## 6. 입찰 버튼 표시 규칙

버튼 텍스트는 현재 로그인한 기사(`DRIVER_UUID`)의 `TB_BUS_RESERVATION` 데이터를 기준으로 결정합니다.

| 조건 | 버튼 텍스트 | 버튼 색상 |
|------|------------|---------|
| `TB_BUS_RESERVATION`에 해당 `REQ_UUID` + `DRIVER_UUID` 데이터 **없음** | **입찰 참여** | Primary (초록) |
| `TB_BUS_RESERVATION`에 데이터 있고, `BID_SEQ` 최대 ROW의 `RES_STAT = 'REQ'` | **입찰 제시 변경** | Secondary (연두/아웃라인) |
| 그 외 상태 (`CONFIRM`, `DONE`, `CANCELLATION_OF_BID` 등) | **입찰 참여** (새 입찰 가능) | Primary (초록) |

### 버튼 판단 SQL 로직

```sql
-- 현재 기사의 해당 요청에 대한 최신 입찰 상태 조회
SELECT RES_STAT
FROM TB_BUS_RESERVATION
WHERE REQ_UUID = UUID_TO_BIN(?)     -- TB_AUCTION_REQ.REQ_UUID
  AND DRIVER_UUID = UUID_TO_BIN(?)  -- 현재 로그인 기사 UUID
ORDER BY BID_SEQ DESC
LIMIT 1;

-- 결과가 없으면    → 「입찰 참여」
-- RES_STAT = 'REQ' → 「입찰 제시 변경」
-- 그 외            → 「입찰 참여」 (신규 입찰 가능 처리)
```

---

## 7. 상태 배지 표시 규칙

- 카드 제목 옆에 **`REQ_STAT` 값을 대문자**로 표시한다 (예: `BIDDING`).
- 스타일: 연한 녹색 배경 + 진한 녹색 글자의 둥근 pill 형태.
- `EXPIRE_DT` 기반의 「실시간 경매 / 마감 임박」 배지는 **사용하지 않는다.**

## 8. REST API

### 8-1. GET /api/auction-list

**역경매 목록 조회 — 기사 입찰 상태 포함**

| 항목 | 내용 |
|------|------|
| **메서드** | `GET` |
| **경로** | `/api/auction-list` |
| **쿼리 파라미터** | `driverUuid` (string, 필수 — 버튼 텍스트·동일출발일 제외) |
| **응답** | JSON — `total` + `items` 배열 |

**동일 출발일 제외**  
기사(`DRIVER_UUID`)에 대해 **다른 견적**(`ar.REQ_UUID <> r.REQ_UUID`)에 `RES_STAT = 'CONFIRM'` 인 예약이 있고, 그 견적의 `START_DT` **날짜**가 `r.START_DT`와 같으면 해당 `r` 는 제외한다.

**SQL 쿼리 (핵심)**

```sql
SELECT ...
FROM TB_AUCTION_REQ r
LEFT JOIN TB_AUCTION_REQ_BUS b ON b.REQ_UUID = r.REQ_UUID
WHERE r.REQ_STAT = 'BIDDING'
  AND NOT EXISTS (
    SELECT 1
      FROM TB_BUS_RESERVATION res2
      INNER JOIN TB_AUCTION_REQ ar ON ar.REQ_UUID = res2.REQ_UUID
     WHERE res2.DRIVER_UUID = UUID_TO_BIN(?)
       AND res2.RES_STAT = 'CONFIRM'
       AND DATE(ar.START_DT) = DATE(r.START_DT)
       AND ar.REQ_UUID <> r.REQ_UUID
  )
GROUP BY ...
-- myBidStat 서브쿼리 및 ORDER BY 동일 (바인딩: driverUuid 2회)
```

**응답 스키마**

- `total === 0` 인 경우 **`emptyMessage`** 필드가 포함될 수 있다. 값은 **「등록된 입찰이 없습니다」** (프론트 기본 문구와 동일).

```json
{
  "total": "number",
  "emptyMessage": "string (optional, total=0 일 때)",
  "items": [
    {
      "reqUuid":       "string (UUID)",
      "tripTitle":     "string | null",
      "startAddr":     "string",
      "endAddr":       "string",
      "startDt":       "datetime",
      "endDt":         "datetime",
      "passengerCnt":  "number",
      "reqStat":       "BIDDING",
      "reqAmt":        "number",
      "expireDt":      "datetime",
      "regDt":         "datetime",
      "busType":       "string | null",
      "busCnt":        "number",
      "myBidStat":     "REQ | CONFIRM | DONE | CANCELLATION_OF_BID | null"
    }
  ]
}
```

> - `myBidStat = null` → 입찰 이력 없음 → **「입찰 참여」**
> - `myBidStat = 'REQ'` → 입찰 중 → **「입찰 제시 변경」**
> - 그 외 값 → 신규 입찰 가능 → **「입찰 참여」**

## 9. 상태 관리 (React State)

| 상태 | 타입 | 설명 |
|------|------|------|
| `items` | `array` | API 응답 목록 전체 |
| `currentIndex` | `number` | 현재 노출 인덱스 (0부터) — **표시는 항상 1건** `slice(n, n+1)` |
| `emptyMessage` | `string` | 빈 목록 안내 — API `emptyMessage` 또는 기본 **「등록된 입찰이 없습니다」** |
| `loading` | `boolean` | 조회 로딩 여부 |
| `loadError` | `string\|null` | 오류 메시지 |
| `selectedReqUuid` | `string\|null` | 「입찰 참여/변경」 클릭 시 → `TravelerQuoteRequestDetails` 모달 호출 트리거 |

### 자동 스크롤 동작

```
useEffect (items 변경 시 currentIndex → 0 리셋)
useEffect (자동 순환 타이머)
  └─ items.length <= 1 이면 타이머 미사용
  └─ items.length >= 2 일 때만:
       setInterval(5000ms) — 5초마다 currentIndex = (currentIndex + 1) % items.length
  └─ 표시: items.slice(currentIndex, currentIndex + 1)
  └─ 카드 key에 인덱스를 포함해 전환 시 슬라이드 애니메이션 재생
```
