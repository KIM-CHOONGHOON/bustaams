# 버스 운행 완료 상세 화면 (BusOperationCompletionDetails)

## 0. 스키마·식별자 기준 (`ARCHITECTURE.md`)

- **공식 DDL:** [`ARCHITECTURE.md`](./ARCHITECTURE.md) 및 동일 정의 복본 [`busTaams_web/BUSTAAMS_테이블 생성 쿼리 전체.md`](./busTaams_web/BUSTAAMS_테이블%20생성%20쿼리%20전체.md) — 본 화면과 직접 연관: **`TB_BUS_RESERVATION`**, **`TB_AUCTION_REQ`**, **`TB_AUCTION_REQ_BUS`**, **`TB_AUCTION_REQ_VIA`**, **`TB_USER`**.
- **UUID → 비즈니스 ID:** 레거시 `RES_UUID` / `REQ_UUID` / `REQ_BUS_UUID` / `DRIVER_UUID`(BINARY) 등과 달리, 목표 스키마는 **`RES_ID`**, **`REQ_ID`**, **`REQ_BUS_ID`**, **`VIA_ID`**, 완료 상태 **`DATA_STAT = 'DONE'`**, 기사 **`DRIVER_ID`**(`TB_BUS_RESERVATION`에 저장되는 기사 식별자 — 업무 규칙상 `TB_USER.USER_ID` 또는 `USER_SEQ_ID` 등과 동일 계열로 맞춤).
- **ID 채번:** [`ARCHITECTURE.md`](./ARCHITECTURE.md) **「비즈니스 식별자(ID) 채번 규칙」** — `RES_ID`, `REQ_ID`, `REQ_BUS_ID`, `VIA_ID` 등 **`VARCHAR(n)` 순번형**은 해당 컬럼 DDL의 자릿수 **`n`**에 맞춰 **왼쪽 0 채움** (`String(seq).padStart(n, '0')`). 예: `VARCHAR(10)`이면 `1` → `'0000000001'`. **`TB_USER.USER_ID`**(로그인 ID·암호문 저장 등)는 **가입·외부 결정 값**으로 순번 규칙 **예외**이며, **`FILE_ID`** 등 다른 규칙은 해당 DDL 주석·§채번 표를 따른다.
- **REST·프론트 권장명:** 쿼리·props·응답은 DB와 맞추어 **`driverId`**(세션 로그인 ID 또는 `TB_USER.USER_ID`와 동일하게 쓰는 문자열), **`resId`**, **`reqId`**, **`reqBusId`**. 레거시 별칭 **`driverUuid`**, **`resUuid`**, **`reqUuid`**, **`reqBusUuid`** 는 JSON·쿼리에서 **동일 값 병행**(`resId`/`resUuid` 등). 값은 **UUID가 아닐 수 있음**(0채움 ID).
- **타입 정합:** 문서상 `TB_BUS_RESERVATION.DRIVER_ID`·`TRAVELER_ID`가 `varchar(10)`이고 `TB_USER.USER_ID`가 `varchar(255)`인 등 **DDL 간 길이가 어긋날 수 있다**. 다른 테이블(예: `TB_SUBSCRIPTION`)은 `DRIVER_ID varchar(255)`로 `TB_USER`를 FK 참조한다. **운영 DB 실제 DDL**에 맞춰 FK·저장 값을 통일하고, API는 **`driverId` 직접 비교**와 **`USER_UUID` 기반 서브쿼리**를 **병행 시도**한다(§4-2·`busOperationCompletionDetails.js`).

### 3-4. 「요청 차량」(`TB_AUCTION_REQ_BUS`)

- API 배열 **`auctionReqBuses`** 를 순서대로 출력.
- 각 행: **차종 라벨**(`BUS_TYPE_CD` → `getVehicleLabel`) + **대수**(`reqBusCnt`) + 행별 금액(**`reqAmtKrw`** — 레거시는 `REQ_AMT`, ARCH는 `REQ_AMT` 또는 `TOLLS_AMT`+`FUEL_COST`).
- JSON **`reqBusId`** (레거시 키 `reqBusUuid`): **ARCH**에서는 **`REQ_BUS_ID`** 문자열(0채움 순번 등); 레거시는 `BIN_TO_UUID(REQ_BUS_UUID)`.
- 배열이 비면 안내 문구 **「등록된 요청 차량이 없습니다.」**

## 4. 백엔드·REST API

| 항목 | 내용 |
|------|------|
| **메서드** | `GET` |
| **경로** | `/api/bus-operation-completion-details` |
| **구현** | `busTaams_server/routes/busOperationCompletionDetails.js` |

### 4-1. 쿼리 파라미터

| 이름 | 필수 | 설명 |
|------|------|------|
| `driverId` | O (권장) | **`TB_USER.USER_ID`** — `TB_BUS_RESERVATION.DRIVER_ID = ?` 로 직접 비교 |
| `driverUuid` | O (레거시) | **UUID 형식이면** `TB_USER.USER_UUID = UUID_TO_BIN(?)` 로 `USER_SEQ_ID`/`USER_ID`를 구해 `DRIVER_ID`와 비교. **UUID가 아니면** `DRIVER_ID = ?` 로 처리(구현체는 `driverUuid` 키에 `USER_ID`를 넣는 패턴 허용) |
| `resId` | 조건 | **ARCH:** `RES_ID`. 레거시 쿼리 키 `resUuid` — **`reqId`와 둘 중 하나 이상** |
| `reqId` | 조건 | **ARCH:** `REQ_ID`. 레거시 `reqUuid` — `resId` 단독 0건 시 **폴백**으로 최신 `DONE` 예약 1건 |

> 구현체(`busOperationCompletionDetails.js`)는 **`driverId` 또는 `driverUuid` 중 하나**, **`resId` 또는 `resUuid`**, **`reqId` 또는 `reqUuid`** 를 수용한다. **`GET /api/bus-operation-completion-list`** 도 **`driverId`·`driverUuid`를 동시에** 받을 수 있으며, ARCH 경로에서 비UUID `driverId`는 `DRIVER_ID = ?`, UUID는 `TB_USER` 서브쿼리로 매칭한다(`busOperationCompletionList.js`).

### 4-2. 스키마 분기·ARCH 조합 시도 순서

1. **레거시 SQL**을 먼저 실행한다(`RES_STAT`, `RES_UUID`/`REQ_UUID`, `DRIVER_UUID`, `USER_UUID` 기반 여행자 조인).
2. **`ER_BAD_FIELD_ERROR`(1054)** 이면 **ARCHITECTURE 경로**로 전환한다: `DATA_STAT='DONE'`, `r.REQ_ID = res.REQ_ID`, `RES_ID`/`REQ_ID` 필터.
3. ARCH 경로에서는 한 번에 고정 SQL이 아니라, 아래 조합을 **스키마 오류가 날 때까지 순차 시도**한다(마이그레이션 중간 DDL 대응).
   - **여행자 요청 금액:** `r.REQ_AMT` 사용 ↔ 마스터에 컬럼이 없으면 **`SUM(TOLLS_AMT+FUEL_COST)`** 서브쿼리(`TB_AUCTION_REQ_BUS`).
   - **기사 매칭:** `USER_UUID` 서브쿼리 ↔ **`DRIVER_ID = ?`** 직접(파라미터 **`driverId`** 또는 UUID가 **아닌** `driverUuid`).
   - **여행자 성명:** `u.USER_ID = res.TRAVELER_ID` ↔ `COALESCE(res.TRAVELER_ID, r.TRAVELER_ID)`(후자 컬럼이 없으면 다음 조합으로 스킵).

### 4-3. 데이터 소스 (요약)

| 구분 | 테이블 | 조건 / 조인 |
|------|--------|-------------|
| 예약·기사·금액 | `TB_BUS_RESERVATION` | **레거시:** `RES_STAT='DONE'`, `DRIVER_UUID`, `RES_UUID`. **ARCH:** `DATA_STAT='DONE'`, `DRIVER_ID`, `RES_ID` |
| 견적 마스터 | `TB_AUCTION_REQ` | **레거시:** `REQ_UUID = res.REQ_UUID`. **ARCH:** `REQ_ID = res.REQ_ID` |
| 여행자 성명 | `TB_USER` | **레거시:** `USER_UUID = COALESCE(res.TRAVELER_UUID, r.TRAVELER_UUID)`. **ARCH:** `USER_ID` = `res.TRAVELER_ID` (또는 요청 마스터 `TRAVELER_ID`와 병합 시도) |
| 요청 차량 | **`TB_AUCTION_REQ_BUS`** | **레거시:** `REQ_UUID`·`REQ_BUS_UUID`·`REQ_BUS_CNT`·`REQ_AMT`. **ARCH:** `REQ_ID`·`REQ_BUS_ID`; `REQ_BUS_CNT`/`REQ_AMT` 없으면 **`TOLLS_AMT`+`FUEL_COST`** |
| 타임라인 | **`TB_AUCTION_REQ_VIA`** | **레거시:** `REQ_UUID`. **ARCH:** `REQ_ID`. `REQ_ID` 컬럼이 없으면 `REQ_UUID` 폴백(이때 `reqId`/`reqUuid`가 UUID 형식일 때만 `UUID_TO_BIN` 사용 — 숫자 `REQ_ID` 오바인딩 방지) |

### 4-4. 응답 필드 (요약)

| JSON 필드 | 출처 |
|-----------|------|
| `travelerName` | `TB_USER.USER_NM` (복호화) |
| `tripTitle` | `TRIP_TITLE` |
| `startDt` | `START_DT` |
| `endDt` | `END_DT` |
| `passengerCnt` | `PASSENGER_CNT` |
| `travelerRequestKrw` | **`REQ_AMT`** 또는 ARCH에서 마스터 컬럼 부재 시 **요청 버스 행 합산** |
| `driverBidKrw` | `TB_BUS_RESERVATION.DRIVER_BIDDING_PRICE` |
| **`auctionReqBuses`** | **`TB_AUCTION_REQ_BUS`** — **`reqBusId`**(권장), 레거시 키 `reqBusUuid`, `busTypeCd`, `reqBusCnt`, `reqAmtKrw`, `sortOrder` |
| `timeline` | `TB_AUCTION_REQ_VIA` |
| `resId`, `reqId` | 식별 — **ARCH:** `RES_ID`, `REQ_ID`(0채움 순번 문자열 등). 레거시 응답 키 `resUuid`/`reqUuid` |

**이 API에서는** `startAddr` / `endAddr`·지도·`finalAmountKrw` 등은 **응답하지 않음**.

### 4-5. `TB_AUCTION_REQ_VIA` — 타임라인 (최대 9행)

`VIA_ORD` **ASC**, `VIA_TYPE`별 `lineText` 규칙은 기존과 동일(`START_NODE`→출발지 … `END_NODE`→도착지).

### 4-6. HTTP `404`

해당 기사·`resId`/`reqId`(레거시 `resUuid`/`reqUuid`) 조합에 완료 건이 없으면 `404` JSON.

## 5. 오류 확인

| 증상 | 점검 |
|------|------|
| **`Unknown column 'VIA_TYPE'`** 등 | `TB_AUCTION_REQ_VIA` 스키마·`VIA_TYPE` 컬럼 |
| **`Cannot GET /api/...`** | 라우트·서버 재시작·`VITE_API_BASE_URL` |
| **ARCH DB에 `USER_UUID` 없음** | 쿼리 **`driverId`=`TB_USER.USER_ID`** 로 호출하거나, `driverUuid` 키에 동일 값을 넣어 서버가 **`DRIVER_ID = ?`** 경로로 조회하게 한다. |
