# BusTaams 로그인 화면 및 백엔드 연동 설계

이 문서는 BusTaams 프로젝트의 로그인 화면 구현 방식과 백엔드 연동 로직, 그리고 로그인 성공 후 권한에 따른 라우팅 흐름을 정의합니다.

## 1. 프론트엔드 화면 구현 (React 전환)

- **원본 소스**: `downloads/bustaams_web/로그인/Login.html`
- **구현 원칙**: 기존 HTML의 화면 형태, 버튼 구성, 색상값, 입력 항목 등 전체적인 시각적 요소를 전혀 훼손하지 않고 React(JSX) 형태로 충실히 재현합니다.
- **컴포넌트 위치**: `src/components/Login/Login.jsx` 및 `Login.css`
- **상태 관리**: 입력받은 아이디(`userId`)와 비밀번호(`password`) 상태를 관리하며, API 통신 후 응답 결과에 따라 부모 컴포넌트(`App.jsx`)로 유저 정보를 전달합니다.

## 2. 데이터베이스 구조 (TB_USER)

최신 회원가입 명세서 및 DB 정책을 따릅니다.

- **주 키 (Primary Key)**: `CUST_ID` (VARCHAR(10), 10자리 숫자 문자열)
- **로그인 아이디**: `USER_ID` (VARCHAR(256), 평문 - 일반 ID 또는 SNS 고유 키값)
- **비밀번호**: `PASSWORD` (Bcrypt 단방향 해시 암호화)
- **기타 정보**: `USER_NM` (성명), `HP_NO` (전화번호), `USER_TYPE` (회원 권한)
- **상태 정보**: `USER_STAT` ('ACTIVE', 'LEAVE', 'BANNED' 등)
- **로그인 타입**: `SNS_TYPE` ('NONE', 'KAKAO', 'NAVER')

## 3. 백엔드 로그인 로직 (RestAPI)

**Endpoint**: `POST /api/users/login`

클라이언트로부터 `userId`와 `password`를 수신하여 다음 검증을 거칩니다:

1. **계정 존재 여부 및 비밀번호 확인**: `USER_ID`로 사용자를 조회하고 `bcrypt.compare`를 통해 비밀번호를 검증합니다.
2. **사용자 상태(USER_STAT) 확인**:
    - `ACTIVE`: 정상 로그인 진행. 응답 데이터에 `CUST_ID`를 포함하여 반환합니다.
    - `ACTIVE`가 아닐 경우: 로그인을 차단하고 각 상태별 사유를 클라이언트에 전달합니다. (예: `LEAVE` - 탈퇴 계정, `BANNED` - 정지 계정 등)
3. **클라이언트 처리**: 서버에서 전달받은 상태 코드가 `ACTIVE`가 아니면, 해당 사유를 **Alert 창**으로 띄워 사용자에게 알립니다.
4. **성공 시 반환**: 로그인 성공 시 프론트엔드 대시보드 표출에 필요한 핵심 정보(`CUST_ID`, `userName`, `userType`, `phoneNo` 등)를 반환합니다.

## 4. 권한 기반 대시보드 라우팅 (App.jsx)

로그인 성공 후 반환받은 `userType`(회원 구분)에 따라 동적으로 메인 화면이 전환됩니다.

1. **TRAVELER (소비자)**
   - `CustomerDashboard` 화면을 호출합니다.
2. **PARTNER (제휴사/파트너)**
   - `PartnerDashboard` 화면을 호출합니다.
3. **DRIVER (기사님)**
   - `DriverDashboard` 화면을 호출합니다.

해당 라우팅 로직은 최상단 컴포넌트인 `App.jsx`의 React 조건부 렌더링에 통합되어 컨트롤됩니다.

## 5. 비밀번호 찾기 프로세스

사용자가 비밀번호를 분실했을 경우 다음의 단계를 통해 계정을 복구합니다.

1. **사용자 정보 입력**: 로그인 화면의 '비밀번호 찾기' 클릭 시 이동하며, `USER_ID`(아이디)와 `HP_NO`(휴대폰 번호)를 입력받습니다.
2. **휴대폰 본인 인증**: 
    - 입력된 정보가 DB의 회원 정보와 일치하는지 확인합니다.
    - 일치할 경우, 해당 번호로 **SMS 인증번호**를 발송하여 본인 확인을 완료합니다.
3. **임시 비밀번호 발송**:
    - 본인 인증 성공 시, 시스템에서 생성한 **임시 비밀번호**를 SMS로 즉시 발송합니다.
    - DB의 해당 계정 비밀번호를 임시 비밀번호(해싱됨)로 즉시 업데이트합니다.
4. **비밀번호 재설정 화면 이동**:
    - 임시 비밀번호 발송 완료 후, 사용자를 **비밀번호 재설정(Change Password) 화면**으로 자동 이동시킵니다.
    - 사용자는 임시 비밀번호로 로그인하기 전, 본인이 원하는 새로운 비밀번호로 변경하도록 유도됩니다.
