# bustaams 로그인 모달 (팝업) 설계 정리

메인 홈페이지에 연동된 **로그인 팝업 (LoginModal)**의 구조 및 디자인 반영 내역입니다. **[`ARCHITECTURE.md` - DESIGN 표준 (Web & App 공통)]** 규칙을 엄격히 준수하도록 기존 폼 형태를 완전히 리팩토링했습니다.

## 1. 컴포넌트 팝업 연동
- **"로그인 화면은 홈페이지 전체를 덮어쓰지 말고 bustaams 홈페이지의 가운데에 pop-up 모달 형식으로 출력되어야 한다"** 라는 명확한 필수 설계 방향을 적용했습니다.
- 기존의 불투명한 꽉 찬 화면 방식을 버리고, 진한 회색조의 반투명 배경(`bg-gray-900/50`, 약한 블러 효과)을 사용해 배경에 위치한 실제 홈페이지 영역이 확실히 투영되도록 설계해 완전한 '팝업' 형태를 구축했습니다. 
- 뒷배경이 스크롤되지 않도록(`overflow='hidden'`) 처리하는 로직을 동시에 적용하여 UI 몰입도를 높였습니다.

## 2. 디자인 표준 (Design System) 반영 내역
- **Elevation & Depth**: 모달 본체에 순백색(`surface-lowest`)을 적용하고, 주위 배경은 블러 처리된 어두운 오버레이(`bg-surface/80`)를 입혀 자연스러운 띄움(Lift) 효과를 보장했습니다.
- **입력창 (Form Elements)**: 기존의 흰색 1px 테두리 입력창을 모두 제거하고, 디자인 원칙에 맞게 바탕보다 살짝 어두운 `surface-container-high` 배경(회색 음영) 구조를 썼습니다. 포커스(Focus) 시에만 살짝 빛나는 테두리가 나타납니다.
- **탭바 (Lists/Tabs)**: 하단 선을 이용한 딱딱한 탭 스위처를 버리고, 백그라운드 전환 방식의 Pill(알약) 형태 버튼 탭으로 최적화해 시스템의 **"구분선 전면 금지(No-Line)"** 규칙을 철저히 지켰습니다.
- **컴포넌트 둥글기 (Border Radius)**: 각지고 날카로운 모서리를 최소화하고 디자인 표준에 맞춰 친근하고 부드러운 `lg`, `xl` 라운딩 수치를 전역 적용했습니다.
- **타이포그래피 및 그라데이션**: 텍스트 서체를 시스템 폰트로 교체하고, 로그인 버튼에 기본 Teal(`Primary`) 색상 그라데이션을 부여해 입체감을 살렸습니다.

## 3. 사용자 상호작용 (Navigation)
- **팝업 열기:** 메인 페이지 상단의 "로그인" 텍스트 클릭 시 반응형 팝업이 등장합니다.
- **팝업 닫기:** 배경(블러 영역)을 클릭하거나 우측 상단 `X` 닫기 버튼을 클릭하면, 부드러운 이펙트 애니메이션(`fade-out`)과 함께 창이 즉각 닫힙니다.
