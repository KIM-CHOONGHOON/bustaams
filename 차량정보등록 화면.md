# 차량정보등록 화면

기사가 보유 버스를 등록·수정하는 모달(`BusInformationSetup`)의 **UI·REST·DB 정본**입니다.  
**컬럼 ID·타입·길이**는 [`BusTaams 테이블.md`](./BusTaams%20테이블.md)를 따릅니다. GCS·파일 정책은 [`ARCHITECTURE.md`](./ARCHITECTURE.md)를 참고합니다.

---

## 0. 식별자 정책 (BusTaams와 동일 명칭)

| JSON / 쿼리 필드 | DB 컬럼 | 타입(정본) | 설명 |
|------------------|---------|------------|------|
| **`userId`** | `TB_USER`.`USER_ID` 또는 동일 문자열의 `CUST_ID` | `USER_ID` varchar(256) / `CUST_ID` varchar(10) | REST 식별자(권장). `userUuid`는 하위 호환 별칭으로만 서버가 수용. |
| **`busId`** | `TB_BUS_DRIVER_VEHICLE`.`BUS_ID` | varchar(10), 0패딩 순번 | 차량 PK. 예전 필드명 `busUuid`는 **삭제**; 서버는 일시적으로 `busUuid` 본문 키를 받을 수 있으나 신규 연동은 **`busId`만** 사용. |
| **`fileId`** | `TB_FILE_MASTER`.`FILE_ID` 및 차량의 `*_FILE_ID` | varchar(20), 0패딩 순번 | 파일·썸네일·서류 참조. 예전 `fileUuid` 쿼리/JSON는 **삭제**; 서버는 쿼리에서 `fileUuid`를 **잠시** 병행 인식할 수 있음. |
| 타 도메인 (차량 API 밖) | `TB_AUCTION_REQ`.`TRAVELER_ID` | varchar(10) | 예전 `TRAVELER_UUID` 아님. |
| | `TB_BUS_RESERVATION`.`DRIVER_ID` | varchar(10) | 예전 `DRIVER_UUID` 아님. |
| | `TB_TRIP_REVIEW`.`WRITER_ID`, `DRIVER_ID` | varchar(10) | 예전 `WRITER_UUID` / `DRIVER_UUID` 아님. |

## 1. 개요

`DriverDashboard` → **「버스 정보」** → `BusInformationSetup` 모달.  
`GET /api/common-codes?grpCd=BUS_TYPE`, `GET|POST|PATCH /api/driver/bus*` 로 `TB_BUS_DRIVER_VEHICLE` · `TB_FILE_MASTER` · `TB_BUS_DRIVER_VEHICLE_FILE_HIST` 와 연동합니다.

---

## 2. 화면 진입 (FrontEnd)

1. 로그인 → `DriverDashboard`  
2. 「버스 정보」 → 모달 오픈  
3. 섹션: 기본정보 → 서비스등급(`BUS_TYPE`) → 편의/ADAS → 검사·서류 3종 → 차량 사진 최대 8장  
4. 제출: 신규 등록 / 수정 완료 메시지 후 모달은 자동 닫지 않음  
5. 세션: `ownerId` = `userId` → `custId` → … 순으로만 읽고, API에는 **`userId`** 만 전달  

**소스:** `busTaams_web/src/components/BusInformationSetup/BusInformationSetup.jsx`  
**문서 뷰어:** `CommonView.jsx` — `fileId` + `userId` (props `fileUuid`는 레거시 폴백)

## 3. REST API (Request / Response)

### 3-1. 공통코드

| | |
|---|---|
| **GET** | `/api/common-codes?grpCd=BUS_TYPE` |
| **Response** | `{ items: [ { dtlCd, cdNmKo, cdFnum, … } ] }` — `dtlCd` = `TB_COMMON_CODE.DTL_CD` |

### 3-2. 차량 조회

| | |
|---|---|
| **GET** | `/api/driver/bus?userId=` (필수) |
| **Response 200** | `{ bus: null }` 또는 아래 객체 |

`bus` 객체 (필드명 = API 용어 / DB):

| 필드 | DB | 타입 |
|------|-----|------|
| `busId` | `BUS_ID` | string(10) |
| `vehicleNo` | `VEHICLE_NO` | string |
| `modelNm` | `MODEL_NM` | string |
| `manufactureYear` | `MANUFACTURE_YEAR` | string |
| `mileage` | `MILEAGE` | number |
| `serviceClass` | `SERVICE_CLASS` = `DTL_CD` | string |
| `amenities` | `AMENITIES` JSON | object |
| `hasAdas` | `HAS_ADAS` | boolean |
| `lastInspectDt` | `LAST_INSPECT_DT` | `YYYY-MM-DD` |
| `insuranceExpDt` | `INSURANCE_EXP_DT` | `YYYY-MM-DD` |
| `bizRegFile` | `BIZ_REG_FILE_ID` → join | `{ fileId, orgFileNm, fileExt }` 또는 `null` |
| `transLicFile` | `TRANS_LIC_FILE_ID` | 동일 |
| `insCertFile` | `INS_CERT_FILE_ID` | 동일 |
| `vehiclePhotoFileIds` | `VEHICLE_PHOTOS_JSON` | `string[]` (각 요소 = `FILE_ID`, 최대 8) |

### 3-3. 차량 신규 등록

| | |
|---|---|
| **POST** | `/api/driver/bus` |
| **Content-Type** | `application/json` |

**Request (주요)**  

| 필드 | 필수 | DB 반영 | 비고 |
|------|------|---------|------|
| `userId` | Y | `USER_ID` | |
| `vehicleNo` | Y | `VEHICLE_NO` | |
| `modelNm` | | `MODEL_NM` | |
| `manufactureYear` | | `MANUFACTURE_YEAR` | |
| `mileage` | | `MILEAGE` | |
| `serviceClass` | Y | `SERVICE_CLASS` | |
| `amenities` | | `AMENITIES` | JSON |
| `hasAdas` | | `HAS_ADAS` | |
| `lastInspectDt` / `insuranceExpDt` | | 날짜 컬럼 | |
| `businessLicenseBase64` … | | 업로드 시 `BIZ_REG_FILE_ID` 등 | data URL |
| `vehiclePhotos` | | `VEHICLE_PHOTOS_JSON` + 이력 | `[{ base64, fileName }]` |

**Response 201**

| 필드 | 설명 |
|------|------|
| `message` | 안내 문구 |
| `busId` | 신규 `BUS_ID` |
| `fileIds` | `{ biz?, trans?, ins? }` 각 값 = `FILE_ID` |
| `vehiclePhotoFileIds` | `FILE_ID` 배열 |

### 3-4. 차량 스칼라 수정

| | |
|---|---|
| **PATCH** | `/api/driver/bus` |

**Request:** `userId`, **`busId`**, + 수정할 스칼라들 (`vehicleNo`, `modelNm`, …)  

**Response:** `{ message }`

### 3-5. 서류만 갱신

| | |
|---|---|
| **PATCH** | `/api/driver/bus/documents` |

**Request:** `userId`, **`busId`**, + 변경한 서류에 대해서만 `…Base64` / `…FileName`  

**Response:** `{ message, fileIds? }`

### 3-6. 사진 목록 전체 치환

| | |
|---|---|
| **PATCH** | `/api/driver/bus/photos` |

**Request:** `userId`, **`busId`**, `vehiclePhotos`: 최대 8개 — 유지-only 항목은 `{ fileId }`, 신규는 `{ dataUrl 또는 base64, fileName }`  

**Response:** `{ message, vehiclePhotoFileIds }`

### 3-7. 파일 스트리밍 / CommonView

| | |
|---|---|
| **GET** | `/api/driver/bus-documents/file?userId=&fileId=` |
| **GET** | `/api/common-view/bus-document/meta?userId=&fileId=` |
| **GET** | `/api/common-view/bus-document/download?userId=&fileId=` |

`fileId` = `TB_FILE_MASTER.FILE_ID`. (일부 환경에서 쿼리 `fileUuid` 잔존 수용 가능 — 신규는 `fileId`만 사용.)
