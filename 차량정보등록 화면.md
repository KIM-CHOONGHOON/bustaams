# 차량정보등록 화면 개발

## 1. 개요
기사님이 자신의 버스 차량 정보를 플랫폼에 등록할 수 있는 전용 모달(`BusInformationSetup`) 개발 내역입니다. 이는 기존에 제공된 HTML 퍼블리싱을 React 구조로 이관하고, 전용 데이터를 처리할 수 있도록 DB 테이블 설계 및 백엔드 연동을 포함하여 구현되었습니다.

## 2. 화면 진입 흐름 (FrontEnd Flow)
1. **드라이버 모드 로그인**: 기사님 계정으로 로그인 후 `DriverDashboard` 진입.
2. **바로가기 메뉴(Quick Menu) 클릭**: 화면에 있는 10개의 버튼 중 **"버스 정보"** 버튼 클릭.
3. **BusInformationSetup 모달 오픈**: 
   - `App.jsx`에 정의된 `showBusInfoModal` 상태를 통해 모달 노출 (`fixed inset-0` 형태로 전체 화면 오버레이).
4. **차량 정보 입력**:
   - `01. 기본 정보`: 차량 번호, 모델명, 연도, 주행거리 등.
   - `02. 서비스 등급`: `GET /api/common-codes?grpCd=BUS_TYPE` 으로 받은 버스 종류 버튼(그리드).
   - `03. 편의 및 사양`: 최신 ADAS 토글 + 어메니티(와이파이·USB 등).
   - `04. 검사 및 서류`: 최근 정기점검일, 보험 만료 예정일, 서류 3종 업로드.
   - `05. 차량 사진`: 최대 8슬롯.
5. **차량 등록·수정 완료**: 하단 버튼 — 신규 「차량 등록 완료」, 기존 차량 「차량 수정 완료」. 처리 중 「처리 중…」. 성공 시 **모달을 닫지 않으며** `DriverDashboard`(버스기사 메인 대시보드)로 돌아가지 않음 — 같은 모달에서 계속 확인·추가 수정 가능. 닫기는 헤더·배경·「취소」로만 수행.

## 3. BackEnd 연동 (API & Database)

### 3-1. API (경로 유지, 내부 저장소만 변경)
| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/common-codes?grpCd=BUS_TYPE` | 버스 종류 공통코드 |
| GET | `/api/driver/bus?userUuid=` | 기사 소유 차량 1건 조회(없으면 신규 UI) |
| POST | `/api/driver/bus` | 차량 신규 등록 |
| PATCH | `/api/driver/bus` | 스칼라 필드 수정 |
| PATCH | `/api/driver/bus/documents` | 서류만 재업로드 |
| PATCH | `/api/driver/bus/photos` | 차량 사진 목록 교체 |
| GET | `/api/driver/bus-documents/file?userUuid=&fileUuid=` | 서류·사진 미리보기 스트리밍 |

프론트는 위 경로를 그대로 사용합니다. 응답 JSON의 **`busUuid`** 는 DB 컬럼 **`TB_BUS_DRIVER_VEHICLE.BUS_ID`** 와 동일한 UUID 문자열입니다(이름만 API에서 `busUuid` 로 노출).

### 3-2. DB 테이블 (단일 마스터: TB_BUS_DRIVER_VEHICLE)

**설계 정리**

- 기사 보유 버스·**ADAS(`HAS_ADAS`)**·편의(`AMENITIES` JSON)·서류 FK·차량 사진 JSON은 모두 **`TB_BUS_DRIVER_VEHICLE`** 에서 관리합니다.
- **`TB_DRIVER_BUS` 는 앱의 주 테이블이 아닙니다.** 예전 스키마(레거시)이며, 과거 데이터가 있을 때만 서버가 **`TB_BUS_DRIVER_VEHICLE`** 생성 후 **`INSERT IGNORE … SELECT`** 로 한 번 복사합니다. **신규·수정·조회는 모두 `TB_BUS_DRIVER_VEHICLE` 만 사용합니다.**
- 이력 테이블은 **`TB_BUS_DRIVER_VEHICLE_FILE_HIST`** 이며, 컬럼 **`BUS_ID`** 가 마스터 PK와 연결됩니다(구 **`TB_DRIVER_BUS_FILE_HIST.BUS_UUID`** 대응).

**주요 컬럼 (마스터)**

| 컬럼 | 설명 |
|------|------|
| `BUS_ID` | PK, BINARY(16) — API `busUuid` 와 동일 값 |
| `USER_UUID` | 소유 기사 |
| `VEHICLE_NO`, `MODEL_NM`, `MANUFACTURE_YEAR`, `MILEAGE` | 기본 정보 |
| `SERVICE_CLASS` | `TB_COMMON_CODE` `BUS_TYPE` 의 `DTL_CD` |
| `AMENITIES` | JSON (wifi, usb, …) |
| **`HAS_ADAS`** | **CHAR(1) `'Y'` / `'N'`** — 최신 ADAS 토글 |
| `LAST_INSPECT_DT`, `INSURANCE_EXP_DT` | 점검·보험 일자 |
| `BIZ_REG_FILE_UUID`, `TRANS_LIC_FILE_UUID`, `INS_CERT_FILE_UUID` | 서류 FK |
| `VEHICLE_PHOTOS_JSON` | 사진 `FILE_UUID` 배열(최대 8) |

**DDL 파일 (수동 실행·참고)**

- `busTaams_server/sql/tb_bus_driver_vehicle.sql` — 마스터 테이블
- `busTaams_server/sql/tb_bus_driver_vehicle_file_hist.sql` — 이력 테이블
- `busTaams_server/sql/tb_driver_bus.sql` — **@deprecated** 레거시 정의 참고용

서버는 `ensureTbBusDriverVehicleSchema`, `ensureTbBusDriverVehicleFileHistTable`, `migrateLegacyTbDriverBusToBusDriverVehicle` (`server.js`) 로 위 스키마를 맞춥니다.

- `busTaams_web/src/components/BusInformationSetup/BusInformationSetup.jsx` — 모달 전체(공통코드·조회·신규/수정·서류·사진).

### 4-3. 서류 파일 뷰어 (CommonView 연동)

각 서류 카드 하단에 **「파일보기」** 버튼이 항상 고정 표시됩니다.

| 항목 | 내용 |
|------|------|
| **대상 서류** | 사업자 등록증 · 운송사업 허가증 · 보험증권 |
| **버튼 위치** | 각 카드 최하단 (업로드 영역·미리보기와 분리된 고정 영역) |
| **활성 조건** | `docSlots[key].mode === 'remote'` — 서버에 이미 저장·등록된 파일이 있을 때만 |
| **비활성 조건 ①** | 해당 서류 슬롯이 비어 있음 (`slot === null`) |
| **비활성 조건 ②** | 파일을 로컬에서 선택했지만 아직 서버에 저장하지 않은 상태 (`slot.mode === 'local'`) |
| **비활성 시 표시** | 버튼 회색·반투명 + `cursor-not-allowed` + 툴팁 「서버에 등록된 파일이 없습니다」 |
| **동작** | `setCommonViewTarget({ fileUuid, docTitle })` → 모달 내 CommonView 렌더링 |
| **권한** | `userUuid`(로그인 기사) + `fileUuid`를 서버가 교차 검증 — 본인 파일만 허용 |

**상태별 버튼 동작 요약:**

| `docSlots[key]` 상태 | `hasRemote` | 파일보기 버튼 |
| :--- | :---: | :--- |
| `null` — 파일 없음 | `false` | ❌ 비활성 (회색, 클릭 불가) |
| `mode: 'local'` — 선택됨, 미전송 | `false` | ❌ 비활성 (회색, 클릭 불가) |
| `mode: 'remote'` — 서버 등록 완료 | `true` | ✅ 활성 (녹색, CommonView 오픈) |

**카드 레이아웃 구조 (`BusInformationSetup.jsx`):**

```
┌─────────────────────────────┐
│   [아이콘]  서류명  힌트      │  ← <label> 클릭 시 파일 업로드 (hidden input)
│   (파일 선택 시: 미리보기     │
│    + 제거 버튼)               │
├─────────────────────────────┤
│  👁 파일보기                  │  ← 항상 고정. remote 파일 있을 때만 활성
└─────────────────────────────┘
```

**구현 위치 (`BusInformationSetup.jsx`):**
- `const [commonViewTarget, setCommonViewTarget] = useState(null)`
- 서류 카드 최하단 `<button disabled={!hasRemote} title={...}>` — `hasRemote = slot?.mode === 'remote'`
- 클릭 시 `setCommonViewTarget({ fileUuid: slot.fileUuid, docTitle: title })`
- 컴포넌트 말미: `{commonViewTarget && <CommonView close={...} fileUuid={...} userUuid={userUuid} docTitle={...} />}`

## 4. 특이사항 및 향후 과제

### 4-1. 제작 연도 (`01. 기본 정보`)

| 항목 | 내용 |
|------|------|
| **저장 형식** | 문자열 `YYYY년형` (예: `2024년형`). API `MANUFACTURE_YEAR` 에 그대로 전달. |
| **선택 범위** | **시스템(당해) 연도 ~ 당해 기준 20년 전**까지. 예: 2026년이면 `2006년형` ~ `2026년형`. (이전 구현은 약 5~6개 연도만 고정 목록이었음.) |
| **UI** | **`<select>`** 로 목록을 위·아래 스크롤해 선택하거나, **「직접 입력」** 필드에 **네 자리 연도(YYYY)** 만 입력. 네 자리가 유효 범위이면 `YYYY년형`으로 반영. |
| **레거시 값** | DB에 범위 밖 연도가 있으면(예: 과거 데이터) 목록에 해당 연도를 **추가**해 표시·저장 가능. 신규 등록만 `yearMin`~`yearMax` 밖이면 제출 시 안내. |

구현: `BusInformationSetup.jsx` — `MANUFACTURE_YEAR_RANGE_BACK = 20`, `parseManufactureYearLabel` / `toManufactureYearLabel`, `manufactureYearOptions`(`useMemo`).

### 4-2. 등록·수정 완료 후 동작

- 성공 시 `alert` 로만 안내하고 **`close()`(모달 닫기)를 호출하지 않음** — 사용자가 대시보드로 “복귀”하지 않고 모달이 유지됨.
- 신규 등록(`POST /api/driver/bus`) 성공 응답의 **`busUuid`** 로 `existingBusUuid` 를 갱신해, 이후 같은 세션에서 **수정 모드**(「차량 수정 완료」)로 이어질 수 있음.

- 모달이 열려 있을 때는 뒷 배경(Dashboard)이 스크롤되지 않도록 `document.body.style.overflow = 'hidden'` 이 적용됩니다.
- 개발 시 API 베이스 URL은 `VITE_API_BASE_URL` 이 비어 있으면 동일 출처 `/api` + Vite 프록시(`vite.config.js` 의 `/api` → 백엔드)를 사용할 수 있습니다.
- 운영 DB에 **`TB_DRIVER_BUS` 만 있고 `TB_BUS_DRIVER_VEHICLE` 이 없던** 경우: 서버가 새 테이블을 생성한 뒤 레거시에서 데이터를 복사합니다. 이후 신규 데이터는 **`TB_BUS_DRIVER_VEHICLE`** 에만 쌓입니다.

## 5. 변경 이력 요약 (TB_DRIVER_BUS → TB_BUS_DRIVER_VEHICLE)

| 항목 | 내용 |
|------|------|
| 마스터 테이블명 | `TB_DRIVER_BUS` → **`TB_BUS_DRIVER_VEHICLE`** |
| PK 컬럼명 | `BUS_UUID` → **`BUS_ID`** (값은 동일한 UUID) |
| ADAS | **`HAS_ADAS`** 는 마스터에 유지 (CHAR(1) Y/N) |
| 이력 테이블 | `TB_DRIVER_BUS_FILE_HIST` → **`TB_BUS_DRIVER_VEHICLE_FILE_HIST`**, FK 컬럼 **`BUS_ID`** |
| API URL | **변경 없음** (`/api/driver/bus` 등) |
| JSON 필드명 | 응답 **`busUuid`** = DB **`BUS_ID`** |

## 6. 신규·연동 테이블 생성 쿼리 (DDL)

차량정보등록 화면과 직접 연관된 **신규 테이블**은 아래 두 개입니다.  
공통코드(`TB_COMMON_CODE`)는 `GET /api/common-codes?grpCd=BUS_TYPE` 에서 사용하며, 서버가 없으면 자동 생성합니다.

> 수동 실행 시 `bustaams` 등 대상 DB에 맞게 적용합니다. `sql/` 폴더의 파일과 동일한 내용입니다.

### 6-1. `TB_BUS_DRIVER_VEHICLE` (기사 보유 버스 마스터 · ADAS 포함)

파일: `busTaams_server/sql/tb_bus_driver_vehicle.sql`

```sql
CREATE TABLE IF NOT EXISTS TB_BUS_DRIVER_VEHICLE (
    BUS_ID BINARY(16) NOT NULL PRIMARY KEY COMMENT '버스 고유 ID',
    USER_UUID BINARY(16) NOT NULL COMMENT '소유 기사 (TB_USER)',
    VEHICLE_NO VARCHAR(20) NOT NULL COMMENT '차량 번호',
    MODEL_NM VARCHAR(100) NOT NULL COMMENT '모델명',
    MANUFACTURE_YEAR VARCHAR(10) NULL COMMENT '제작 연도(표시 문자열)',
    MILEAGE INT DEFAULT 0 COMMENT '주행거리 km',
    SERVICE_CLASS VARCHAR(50) NOT NULL COMMENT 'TB_COMMON_CODE BUS_TYPE DTL_CD',
    AMENITIES JSON NULL COMMENT '편의 옵션 JSON (wifi, usb, …)',
    HAS_ADAS CHAR(1) NOT NULL DEFAULT 'N' COMMENT 'ADAS Y/N',
    LAST_INSPECT_DT DATE NULL COMMENT '최근 정기점검일',
    INSURANCE_EXP_DT DATE NULL COMMENT '보험 만료 예정일',
    BIZ_REG_FILE_UUID BINARY(16) NULL,
    TRANS_LIC_FILE_UUID BINARY(16) NULL,
    INS_CERT_FILE_UUID BINARY(16) NULL,
    VEHICLE_PHOTOS_JSON JSON NULL COMMENT '차량 사진 FILE_UUID 배열(최대 8)',
    REG_DT DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY UK_VEHICLE_NO (VEHICLE_NO),
    KEY IDX_BUS_DRIVER_USER (USER_UUID)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
COMMENT='기사 보유 차량·ADAS·서류';
```

### 6-2. `TB_BUS_DRIVER_VEHICLE_FILE_HIST` (서류·차량 사진 이력)

파일: `busTaams_server/sql/tb_bus_driver_vehicle_file_hist.sql`  
선행: `TB_FILE_MASTER`, `TB_BUS_DRIVER_VEHICLE`

```sql
CREATE TABLE IF NOT EXISTS TB_BUS_DRIVER_VEHICLE_FILE_HIST (
    HIST_UUID BINARY(16) NOT NULL PRIMARY KEY,
    BUS_ID BINARY(16) NOT NULL COMMENT 'FK TB_BUS_DRIVER_VEHICLE.BUS_ID',
    FILE_UUID BINARY(16) NOT NULL,
    FILE_CATEGORY VARCHAR(50) NOT NULL,
    REG_DT DATETIME DEFAULT CURRENT_TIMESTAMP,
    KEY IDX_BUS_FILE (BUS_ID, FILE_UUID),
    KEY IDX_FILE_UUID (FILE_UUID)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 6-3. `TB_COMMON_CODE` (버스 종류 `BUS_TYPE` 등 — 연동)

서버 `ensureTbCommonCodeTable` 와 동일. 최초 1회 생성 후 `BUS_TYPE` 행은 API 호출 시 시드될 수 있음.

```sql
CREATE TABLE IF NOT EXISTS TB_COMMON_CODE (
    GRP_CD VARCHAR(50) NOT NULL,
    DTL_CD VARCHAR(50) NOT NULL,
    CD_NM_KO VARCHAR(200) NOT NULL DEFAULT '',
    CD_DESC VARCHAR(500) NULL,
    USE_YN CHAR(1) NOT NULL DEFAULT 'Y',
    DISP_ORD INT NOT NULL DEFAULT 0,
    PRIMARY KEY (GRP_CD, DTL_CD),
    KEY IDX_COMMON_GRP_USE_ORD (GRP_CD, USE_YN, DISP_ORD)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 8. 파일 업로드 허용 형식 및 제한

### 8-1. 서류 첨부 (사업자 등록증·운송 사업 허가증·보험 증권)

| 항목 | 내용 |
| :--- | :--- |
| **허용 MIME** | `image/jpeg`, `image/png`, `image/webp`, `image/gif`, `application/pdf` |
| **`<input accept>` 값** | `ACCEPT_DOC` 상수 — `image/jpeg,image/png,image/webp,image/gif,application/pdf` |
| **최대 크기** | 단일 파일 **10 MB** |
| **프론트 검증** | `setDocFile` 내 `ALLOWED_DOC_TYPES.includes(file.type)` + 크기 비교 |
| **비허용 시** | `alert` 안내 후 파일 선택 취소 |

### 8-2. 차량 사진

| 항목 | 내용 |
| :--- | :--- |
| **허용 MIME** | `image/jpeg`, `image/png`, `image/webp`, `image/gif` |
| **`<input accept>` 값** | `ACCEPT_IMAGE` 상수 — `image/jpeg,image/png,image/webp,image/gif` |
| **최대 크기** | 단일 파일 **10 MB** |
| **프론트 검증** | `setPhotoAt` 내 `ALLOWED_IMAGE_TYPES.includes(file.type)` + 크기 비교 |
| **비허용 시** | `alert` 안내 후 파일 선택 취소 |

### 8-3. 서버 처리 (`parseDataUrlPayload`)

- `pdf`, `jpg`, `png`, `webp`, `gif` 5가지 확장자를 명시 인식 → GCS 객체 키에 올바른 확장자 지정.
- 그 외 MIME은 `ext = 'bin'` 처리 (프론트 검증 통과 시 도달하지 않음).

### 8-4. CommonView 뷰어 호환성

| 파일 형식 | 인라인 뷰어 | 설명 |
| :--- | :---: | :--- |
| PDF | ✅ | `<iframe>` 렌더링 |
| JPG / JPEG / PNG / WEBP / GIF | ✅ | `<img>` 렌더링 |
| 기타 (BIN 등) | ❌ | 「미리보기 불가 형식」 안내 + 다운로드 버튼 |

> 허용 업로드 형식(PDF·JPG·PNG·WEBP·GIF)은 모두 CommonView에서 인라인 뷰어로 표시 가능합니다.
