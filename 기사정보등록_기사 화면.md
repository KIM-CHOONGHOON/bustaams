# 기사정보등록 (Driver Profile Setup) 화면 명세

본 문서는 `busTaams` 프로젝트의 기사 정보 등록 화면에 대한 프론트엔드 구조, 데이터 매핑 및 백엔드 API 연동 방식을 정의합니다.

## 1. 화면 개요
- **목적:** 회원 가입 후 기사님의 전문성 검증을 위한 추가 정보(면허, 자격증, 프로필 등) 수집
- **접근 경로:** 드라이버 대시보드 > 바로가기 메뉴 > "기사 정보" 버튼 클릭
- **디자인 정책:** 'The Radiant Traveler' 가이드라인 준수 (Tonal Layering, Glassmorphism, Material Symbols 사용)

## 2. 프론트엔드 구현 (`DriverProfileSetup.jsx`)

### 2-1. 주요 컴포넌트 구조
- **Step Navigation:** 왼쪽 사이드바에서 등록 진행 단계(인적사항 → 면허정보 → 자격정보) 시각화
- **Interactive Form:** Tailwind CSS 기반의 반응형 입력 폼
- **File Upload:** 프로필 사진 및 자격증 사본(이미지/PDF) 드래그 앤 드롭 및 업로드 기능

### 2-2. 데이터 상태 (State)
| 필드명 | 설명 | 비고 |
| :--- | :--- | :--- |
| `name` | 기사 성명 | `TB_USER` 기본 데이터와 동기화 |
| `rrn` | 주민등록번호 | 6자리-7자리 입력 후 백엔드에서 암호화 |
| `licenseType` | 면허 종류 | 1종 대형, 1종 보통 등 선택 |
| `licenseNo` | 면허 번호 | 00-00-000000-00 형식 |
| `qualCertNo` | 버스운전 자격번호 | 전문 자격 번호 |
| `bioText` | 자기소개 | 상세 경력 및 강점 서술 |
| `profilePhoto` | 프로필 사진 | Base64 형식으로 임시 저장 후 서버 전송 |
| `qualCert` | 자격증 사본 | Base64 형식으로 임시 저장 후 서버 전송 |

## 3. 백엔드 API 명세

### 3-1. 기사 프로필 저장 (`POST /api/driver/profile-setup`)
- **기능:** 기사 정보 저장 및 관련 파일 GCS 업로드
- **보안:** `RRN_ENC`는 `AES-256-GCM` 알고리즘으로 양방향 암호화하여 저장

**Request Body:**
```json
{
  "userUuid": "...",         // 회원 UUID (BIN_TO_UUID 대응)
  "rrn": "900101-1234567",   // 주민번호 (서버에서 암호화)
  "licenseType": "1종 대형",
  "licenseNo": "12-34-567890-11",
  "licenseIssueDt": "2023-01-01",
  "licenseExpiryDt": "2033-01-01",
  "qualCertNo": "12345678",
  "bioText": "...",
  "profilePhotoBase64": "data:image/png;base64,...",
  "qualCertBase64": "data:image/png;base64,..."
}
```

**처리 로직:**
1. `profilePhotoBase64` 존재 시 GCS `profiles/` 폴더 업로드 후 `TB_FILE_MASTER` 등록
2. `qualCertBase64` 존재 시 GCS `certificates/` 폴더 업로드 후 `TB_FILE_MASTER` 등록
3. `TB_DRIVER_INFO` 테이블에 `INSERT ... ON DUPLICATE KEY UPDATE` 수행 (Upsert)
4. 모든 과정은 데이터 무결성을 위해 단일 트랜잭션으로 처리

## 4. 데이터베이스 매핑 (`TB_DRIVER_INFO`)
- 이 화면에서 입력된 모든 정보는 `BusTaams_Project 테이블 설계.md`의 **4. 기사 전문 정보 관리** 섹션에 정의된 테이블에 저장됩니다.
