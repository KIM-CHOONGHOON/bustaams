# 백엔드 프로젝트 초기화 및 필수 모듈 설치 완료 내역

이 문서는 `busTaams_server` (Node.js 백엔드) 프로젝트 초기 셋업 과정에서 설치된 필수 패키지들과 그 역할을 정리한 문서입니다.

## 1. 프로젝트 초기화 (Init)
명령어 `npm init -y`를 통해 빈 폴더였던 `busTaams_server`를 정식 Node.js 패키지로 선언하고, 필수 메타데이터가 담긴 `package.json` 파일을 자동 생성했습니다.

## 2. 코어 모듈 (Dependencies) 설치 목록
프론트엔드(`busTaams_web`)와 원활하게 통신하고 클라우드 데이터베이스(`cloud-sql-proxy`)에 접속하기 위해 다음 핵심 패키지들이 설치되었습니다. (`npm install express cors dotenv mysql2`)

- **`express`**: Node.js 환경에서 가장 널리 쓰이는 표준 웹 프레임워크입니다. 회원가입, 입찰 조회 등 REST API 라우팅을 쉽고 안전하게 구축할 수 있게 해줍니다.
- **`cors`**: 프론트엔드(`localhost:5173`)와 백엔드 서버 간의 도메인(포트)이 달라서 발생하는 보안 문제(CORS 에러)를 해결하여 원활한 통신을 허용합니다.
- **`dotenv`**: 데이터베이스 접속 비밀번호나 API 키 등 깃허브에 올라가면 안 되는 민감한 환경 변수들을 `.env` 파일로 안전하게 별도 관리할 수 있게 해줍니다.
- **`mysql2`**: 이전에 사용하시려던 Java 기반 연결을 대체하여, 현재 연결 대기 중인 `cloud-sql-proxy` (127.0.0.1:3306) MySQL 데이터베이스와 백엔드 서버를 연동하는 초고속 공식 드라이버입니다.

## 3. 개발자 도구 (DevDependencies) 설치 목록
서버 개발을 편하게 하기 위한 유틸리티를 설치했습니다. (`npm install --save-dev nodemon`)

- **`nodemon`**: 백엔드 코드를 수정하고 저장할 때마다 매번 터미널에서 서버를 껐다 켤 필요 없이, 자동으로 감지하여 서버를 재시작해 주는 필수 개발 자동화 도구입니다.

### 👉 향후 과제
- `server.js` (또는 `app.js`) 메인 서버 구동 파일 생성
- 설치한 모듈들을 불러와 기초적인 `Hello World` 통신 테스트
- `package.json`의 `scripts` 에 `"dev": "nodemon server.js"` 명령어 등록
