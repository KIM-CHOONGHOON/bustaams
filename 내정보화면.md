# 내 정보 관리 화면 (My Profile Modal) 설계서

## 1. 개요
- **목적**: 사용자의 기본 인적 사항 조회 및 수정, 비밀번호 변경, SNS 연동 상태 관리.
- **형태**: 메인 대시보드에서 접근 가능한 **중앙 팝업 모달**.
- **대상**: 모든 사용자 유형 (여행자, 기사, 영업사원).

## 2. UI/UX 디자인 가이드
- **레이아웃**: 
    - 화면 중앙 정렬, 최대 너비 1000px 내외.
    - 상단: 타이틀 및 닫기 버튼.
    - 좌측/우측 분할 또는 섹션별 수직 나열 (스크롤 가능).
- **스타일**:
    - `backdrop-blur`를 적용한 반투명 배경.
    - `surface-container-lowest` 배경색의 카드 UI.
    - `Kinetic Gradient`를 적용한 저장 버튼.
- **인터랙션**:
    - 입력 필드 Focus 시 테두리 하이라이트.
    - 저장 성공 시 토스트 메시지 또는 상단 알림 표시.

### 3. 주요 기능 상세

#### A. 기본 인적 사항 수정
*   **이름**: 실명 확인 용도로 사용 (수정 가능).
*   **이메일**: 계정 알림 및 비밀번호 찾기 등에 활용.
*   **휴대폰 번호 (보안 강화)**:
    *   번호 변경 시 **'변경 인증'** 필수.
    *   인증번호 6자리 발송 및 3분 내 확인 절차 필요 (회원가입과 동일 로직).
    *   기존 번호와 다른 상태에서 인증을 받지 않으면 **'변경사항 저장하기'가 차단**됨.

#### B. 보안 설정 (비밀번호 변경)
*   정보 수정을 위해서는 반드시 **현재 비밀번호**를 입력해야 함 (본인 확인).
*   새 비밀번호 입력 시 비밀번호 확인 칸과 일치 여부를 검증.

### 4. 기술 사양 (API 연동)

| 기능 | 엔드포인트 | 메소드 | 주요 파라미터 |
| :--- | :--- | :--- | :--- |
| 프로필 수정 | `/api/user/profile` | `PUT` | `custId`, `email`, `phoneNo`, `currentPassword`, `newPassword` |
| SMS 발송 | `/api/auth/send-sms` | `POST` | `phoneNo` |
| SMS 검증 | `/api/auth/verify-sms` | `POST` | `phoneNo`, `code` |

## 5. 데이터베이스 연동 (참조)
- **대상 테이블**: `TB_USER`
- **조회/수정 컬럼**:
    - `CUST_ID`: 고유 식별자
    - `USER_ID`: 이메일/로그인 ID
    - `USER_NM`: 이름
    - `HP_NO`: 휴대폰 번호
    - `PASSWORD`: 비밀번호 (암호화 저장)
